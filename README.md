# Luban-GUI

**Luban-GUI** 是 [Luban](https://github.com/focus-creative-games/luban) 命令行导表工具的可视化封装，让策划、测试等非技术用户无需学习命令行，即可完成复杂的表格导出任务。

## ✨ 功能特性

### v0.2（进行中，已完成大部分）

- **📁 项目管理** — 新建/打开项目，多项目快速切换，自动初始化 Luban 工作区
- **📋 表格浏览** — 左侧树状列表展示所有 Excel 表，支持实时名称过滤与导表状态标记
- **👁️ 内容预览** — 右侧面板渲染表格数据，支持列排序、虚拟化渲染（万行不卡顿）
- **🛠️ 快捷创建** — 已稳定支持新建表格、枚举、Bean，并写入对应元数据表
- **🧩 类型系统增强** — 支持常用容器类型（`list` / `set` / `map`）表达；新建类型可即时用于字段选择
- **🧭 预览精准定位** — 预览单元格右键可定位并打开 Excel 目标单元格
- **🗑️ 删除表格闭环** — 支持“仅移除注册”与“同时删除物理文件”两种策略，可记住默认策略
- **▶ 一键全量导表** — `F5` 启动 Luban 进程，完成完整的表格导出；`Ctrl+,` 管理导出配置
- **📋 浮动日志窗口** — `Ctrl+L` 打开独立日志窗口，按级别着色，支持 10,000+ 条日志流畅滚动
- **✅ 配置校验** — `F6` 一键检查导出配置，导表前发现问题
- **💾 配置持久化** — 项目配置自动保存，下次启动直接一键导表

### v0.2 持续完善项

- 枚举 / Bean 命名规则的异常场景提示仍在补充
- 非法容器表达式的字段级拦截提示仍在补充
- 数据类型总览中的来源分类（内置 / 自定义）展示细节仍在补齐

## 📥 安装

### 方式一：下载预编译版本（推荐）

1. 前往 [GitHub Releases](../../releases) 页面下载最新版本
2. 解压 `luban-gui-v0.1-windows-x64.zip`
3. 双击 `LubanGui.exe` 即可运行，**无需安装 .NET 运行时**（self-contained 版本）

**系统要求**：Windows 10 / 11 x64

### 方式二：从源码构建

**前置要求**：[.NET 8 SDK](https://dotnet.microsoft.com/download) 或更高版本

```bash
# 克隆仓库
git clone https://github.com/<your-username>/Luban-GUI.git
cd Luban-GUI

# 运行 GUI（同时会自动构建内嵌的 Luban CLI）
dotnet run --project .\LubanGui\LubanGui.csproj

# 或打包为独立可执行文件
dotnet publish .\LubanGui\LubanGui.csproj -c Release -o .\publish
```

> **说明**：项目内已内嵌 Luban CLI，无需单独安装 Luban。构建时 MSBuild 会自动编译 `lubanSrc/` 并将产物复制到 GUI 的 `bin/luban/` 目录。

## 📖 使用指南

### 第一步：创建或打开项目

启动后，在顶部项目切换栏点击 **「+ 新建项目」**，填写项目名称并选择工作区根目录。GUI 会自动创建以下初始文件结构：

```
<WorkspaceRoot>/<ProjectName>/
├── projectConfig.json        # GUI 导出配置
├── luban.conf                # Luban 全局配置
└── Datas/
    ├── __tables__.xlsx       # 表格元数据
    ├── __beans__.xlsx        # Bean 元数据
    └── __enums__.xlsx        # 枚举元数据
```

若已有 Luban 项目目录，点击 **「📂 打开项目文件夹」** 选择对应文件夹即可。下次启动会自动加载上次打开的项目。

### 第二步：创建表格

点击快捷操作栏的 **「+ 新建表格」**，在弹出对话框中填写：

| 字段 | 说明 |
|------|------|
| 表格名称 | 即 Luban 中的 `full_name`，如 `item.ItemConfig` |
| 字段列表 | 字段名 + 数据类型，第一行默认为 `id (int)` |

确认后，GUI 会自动生成符合 Luban 规范的 `.xlsx` 文件（含 `##var`/`##type`/`##` 元数据行），并注册到 `__tables__.xlsx`。

目前 **「+ 新建枚举」** 和 **「+ 新建Bean」** 已提供入口与基础流程，但仍处于持续完善阶段，建议先以「新建表格」为主流程。

若要使用已有的 xlsx 文件，点击 **「📥 导入文件」** 选择目标文件进行注册。

### 第三步：填写数据

双击左侧表格列表中的任意表格名，右侧面板会预览当前数据。点击列标题上方的 **「↗ 在编辑器中打开」**（或直接双击列标题），会用系统默认 Excel 应用打开对应的 `.xlsx` 文件，直接编辑后保存即可。

### 第四步：配置导出参数

按 **`Ctrl+,`** 或点击菜单 **操作 → 导出配置**，打开导出配置窗口：

| 配置项 | 说明 |
|--------|------|
| 导出目标 | 从 `luban.conf` 动态读取，按需选择 |
| 数据格式 | `bin` / `json` / `lua` 等 |
| 数据输出路径 | 导出数据文件的目标目录 |
| 启用代码导出 | 勾选后填写语言类型、输出路径、命名空间 |

配置会自动保存到 `projectConfig.json`，下次直接导表无需重填。

### 第五步：一键导表

按 **`F5`** 或点击状态栏的 **「▶ 全量导表」** 按钮，启动导表流程。按 **`Ctrl+L`** 打开日志窗口，实时观察导表输出。

- **绿色 SUCCESS** — 导表完成，状态栏更新耗时
- **黄色 WARN** — 有警告（如可选字段缺失），导表仍成功
- **红色 ERROR** — 导表失败，检查 xlsx 格式或路径配置

## 🚀 开发者构建

IDE：Visual Studio 2022 / JetBrains Rider（推荐安装 Avalonia 扩展以获得 AXAML 设计器支持）

```bash
# 完整构建解决方案（含 Luban CLI）

dotnet build .\lubanSrc\Luban.sln

# 发布
dotnet publish .\LubanGui\LubanGui.csproj -c Release -o .\publish
```

## 🏗️ 项目结构

```
Luban-GUI/
├── LubanGui/          # Avalonia 桌面前端（C# + XAML）
├── lubanSrc/          # Luban CLI 工具链源码（完整子项目）
├── docs/              # 设计文档与规划
└── LubanGui.slnx      # 根解决方案（仅包含 GUI 项目）
```

> GUI 不直接引用 Luban 项目。构建时会先编译 `lubanSrc/Luban/Luban.csproj`，再将 CLI 产物复制到 `LubanGui/bin/.../luban/` 目录，作为子进程调用。

### 项目工作区结构

每个 Luban-GUI 项目对应磁盘上如下结构（由 GUI 自动初始化和维护）：

```
<WorkspaceRoot>/
└── <ProjectName>/
    ├── projectConfig.json        # GUI 专属导出配置
    ├── luban.conf                # Luban 全局配置（由 GUI 自动维护）
    ├── Defines/                  # 自定义类型定义
    └── Datas/
        ├── __tables__.xlsx       # 表格元数据（由 GUI 维护）
        ├── __beans__.xlsx        # Bean 元数据（由 GUI 维护）
        ├── __enums__.xlsx        # 枚举元数据（由 GUI 维护）
        └── *.xlsx                # 用户数据表
```

## 🏛️ 架构

LubanGui 采用**五层架构**，层间单向向下依赖：

```
┌──────────────────────────────────────────────────────┐
│               表现层 (Avalonia UI)                   │
│  主窗口 · 项目切换 · 快捷操作 · 导出配置 · 日志窗口   │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│              业务逻辑层 (Service)                     │
│  ProjectManager · SchemaService                      │
│  ExportService · TablePreviewService                 │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│                工具层 (Executor / IO)                 │
│  LubanExecutor · LubanCommandBuilder                 │
│  ExcelWriter · FileOpenService                       │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│              基础设施层 (Infrastructure)              │
│  AppConfigManager · ProjectConfigManager · Logger    │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│           Luban 源适配层 (LubanAdapter)               │
│  LubanSchemaReader · LubanConfAdapter                │
│  LubanTypeMapper                                     │
└──────────────────────────────────────────────────────┘
```

> **Luban 源适配层**是 GUI 与 `lubanSrc\` 之间的唯一接触点，上方各层不直接引用 Luban 源码中的类型。
