## 产品概述

### 定位

**Luban-GUI** 是一个 Windows 桌面应用，为 Luban 表格导出工具提供可视化界面。

**核心目标**: 让非技术用户能够无需命令行即可完成表格导出任务，同时为程序员提供调试友好的日志反馈。

### 解决的核心问题

| 问题 | 原因 | 解决方案 |
|-----|------|--------|
| 命令行操作复杂 | Luban 是 CLI 工具 | 图形界面封装 |
| 参数容易出错 | 参数众多且互相关联 | 配置模板和验证 |
| 执行过程不透明 | 不知道进度如何 | 实时日志窗口 |
| 同样的导表重复配置 | 每次都要输入参数 | 配置保存和加载 |

### 目标用户

- **策划**: 需要快速导表，无需学习命令行
- **程序员**: 集成流程中需要导表，需要清晰的日志用于调试
- **测试**: 验证表格导出，需要清晰的成功/失败提示

## 需求分析

### 用户故事

#### 故事1: 快速导表
**As a** 策划  
**I want to** 不学习命令行，直接点击导表  
**So that** 我能快速完成工作，不被技术细节困扰

**验收条件**:
- UI清晰，核心功能一目了然

- 成功和失败有清晰的视觉反馈

#### 故事2: 快速定位导表错误
**As a** 程序员

- 日志窗口显示所有 stdout 和 stderr
- 错误信息能复制粘贴
- 错误消息清晰易读

#### 故事3: 管理多个导表配置
**I want to** 为不同项目保存不同的配置

**验收条件**:

## 功能设计

### MVP 版本 (v1.0) - 核心功能

#### ✅ F1: 配置管理

**描述**: 用户可以配置 Luban 导表的基本参数，覆盖 Luban CLI 提供的大部分选项。

**配置项**:
```
┌─────────────────────────────────────────────┐
│  导表配置                                   │
├──────────────── 必填 ────────────────────────┤
│  • Luban 可执行文件路径                     │
│  • 配置文件路径 (--conf)                    │
│  • 目标名称 (--target)                      │
├──────────────── 常用可选 ────────────────────┤
│  • 代码输出目标 (--codeTarget, 可多个)      │
│  • 数据输出目标 (--dataTarget, 可多个)      │
│  • 扩展参数 (--xargs, 可多个, 格式 key=val) │
├──────────────── 高级可选 ────────────────────┤
│  • Schema 收集器 (--schemaCollector)        │
│  • Pipeline 名称 (--pipeline)               │
│  • 只输出指定表 (--outputTable, 可多个)     │
│  • 包含标签 (--includeTag, 可多个)          │
│  • 排除标签 (--excludeTag, 可多个)          │
│  • 字段变体 (--variant, 可多个, 格式 k=v)  │
│  • 时区 (--timeZone)                        │
│  • 自定义模板目录 (--customTemplateDir)     │
│  • 验证失败作为错误 (--validationFailAsError)│
│  • NLog 配置文件 (--logConfig)              │
│  • 监听目录自动重生成 (--watchDir, 可多个)  │
│  • 强制加载表数据 (--forceLoadTableDatas)   │
│  • Verbose 输出 (--verbose)                 │
├──────────────── GUI 专有 ────────────────────┤
│  • 配置名称（本地标识，不传给 CLI）         │
└─────────────────────────────────────────────┘
```

**特性**:
- 🔄 显示上次保存的配置
- 📂 文件/文件夹浏览对话框
- ✅ 配置有效性检查（路径存在、必填项非空）
- 💾 自动保存到本地文件（用户首次成功导表后）

#### ✅ F2: 全量导表 (核心功能)

**描述**: 用户点击按钮，调用 Luban 完成一次完整的表格导出。

**流程**:
```
1. 用户点击【全量导表】
   ↓
2. 验证配置（路径存在、文件存在）
   ↓
3. 启动 Luban.exe 子进程
   ↓
4. 异步读取 stdout/stderr，实时显示日志
   ↓
5. 进程完成，统计耗时
   ↓
6. 显示 ✓ 成功/✗ 失败
```

**核心要求**:
- 🔥 **实时日志显示**: 异步读取进程输出，不阻塞 UI
- 🔄 **支持取消**: 用户可随时点击【取消】中止导表
- ⚠️ **异常处理**: 进程出错时清晰显示 stderr
- ⏱️ **显示耗时**: 记录导表花费的时间

**Luban 参数构建** (示例):
```bash
luban.exe \
  --conf "D:\GameData\luban.conf" \
  --target client \
  --codeTarget cs-bin \
  --dataTarget bin \
  --xargs "outputCodeDir=D:\GameData\Gen" \
  --xargs "outputDataDir=D:\GameData\Output"
```

#### ✅ F3: 实时日志窗口 (体验核心🔥)

**描述**: 在窗口内实时展示 Luban 进程的执行日志。

**特性**:
- 📝 逐行显示 stdout 和 stderr（带时间戳），前缀区分
- 🎨 按日志级别着色：
  - `[INFO]` - 灰色（一般信息）
  - `[SUCCESS]` - 绿色（成功）
  - `[ERROR]` - 红色（错误）
  - `[WARN]` - 黄色（警告）
- 📋 自动滚动显示最新日志
- 💾 支持全选和复制，便于用户分享错误信息
- 🧹 自动清空（每次导表时清空）

**性能要求**:
- 支持显示 10000+ 条日志不卡顿
- 内存占用 < 100MB
- 使用虚拟化列表（只渲染可见区域）

**示例日志**:
```
[INFO] 开始导表...
[INFO] 验证配置...
[OUT] Processing Excel files...
[OUT] Sheet1: 100 rows
[OUT] Sheet2: 50 rows
[OUT] Generating C# code...
[SUCCESS] Export completed in 2.34s
```

#### ✅ F4: 配置校验（辅助功能）

**描述**: 用户点击按钮验证配置是否有效。

**校验项**:
- ✓ Luban 可执行文件存在
- ✓ 配置文件 (--conf) 路径存在
- ✓ 目标名称 (--target) 不为空
- ✓ 自定义模板目录（若填写）路径存在
- ✓ 监听目录（若填写）路径存在
- ✓ xargs 格式合法（每项均为 `key=value` 格式）
- ✓ variant 格式合法（每项均为 `key=value` 格式）

**用户体验**:
```
用户点击【校验】
  ↓
有效: 显示 ✓ 配置有效，显示绿色
无效: 显示 ✗ 配置错误，列举所有错误
```

## 用户交互流

### 核心场景1: 首次使用

```
1. 用户启动应用
   ↓
2. 看到空白的配置表单
   ↓
3. 依次点击【浏览】按钮选择：
   • Excel 目录
   • 输出目录
   • Luban 路径
   ↓
4. 点击【校验】按钮
   → 显示 ✓ 配置有效
   ↓
5. 点击【全量导表】
   → 日志窗口显示：
     [INFO] 验证配置...
     [INFO] 启动Luban进程...
     [OUT] Processing...
     [SUCCESS] 导表完成 (耗时 2.34s)
   → 显示 ✓ 成功
   ↓
6. 关闭应用
   ↓
7. 下次启动时
   → 配置被记住
   → 用户可直接点击【全量导表】
```

### 核心场景2: 导表失败

```
1. 用户点击【全量导表】
   ↓
2. Luban 因表格错误失败
   → 日志显示：
     [OUT] Processing...
     [ERR] Invalid column type at Sheet!A1
     [ERR] Exit code: 1
   ↓
3. UI 显示 ✗ 失败
   ↓
4. 用户复制日志，根据错误信息修改 Excel
   ↓
5. 重新点击【全量导表】
```

### 核心场景3: 用户取消

```
1. 用户点击【全量导表】，导表进行中
   ↓
2. 用户改主意，点击【取消】
   → Luban 进程被中止
   → 日志显示 [WARN] 用户中止
   ↓
3. UI 返回就绪状态
```

## 数据模型

### ExportConfig (导表配置)

```csharp
public class ExportConfig
{
    // ── GUI 专有 ──────────────────────────────────────────
    
    /// <summary>
    /// 配置名称（仅用于 GUI 内部区分，不传给 Luban CLI）
    /// 例: GameProject, ToolsProject
    /// </summary>
    public string ProfileName { get; set; } = "Default";

    /// <summary>
    /// Luban 可执行文件路径
    /// 例: C:\Tools\Luban\luban.exe
    /// </summary>
    public string LubanPath { get; set; }

    // ── 必填参数 ──────────────────────────────────────────

    /// <summary>
    /// Luban 配置文件路径 (--conf)
    /// 例: D:\GameData\luban.conf
    /// </summary>
    public string ConfFile { get; set; }

    /// <summary>
    /// 生成目标名称 (--target / -t)
    /// 例: client, server, all
    /// </summary>
    public string Target { get; set; }

    // ── 常用可选参数 ──────────────────────────────────────

    /// <summary>
    /// 代码输出目标列表 (--codeTarget / -c, 可多个)
    /// 例: ["cs-bin", "typescript-json"]
    /// </summary>
    public List<string> CodeTargets { get; set; } = new();

    /// <summary>
    /// 数据输出目标列表 (--dataTarget / -d, 可多个)
    /// 例: ["bin", "json"]
    /// </summary>
    public List<string> DataTargets { get; set; } = new();

    /// <summary>
    /// 扩展参数列表 (--xargs / -x, 格式: key=value, 可多个)
    /// 例: ["outputCodeDir=D:\\Gen", "outputDataDir=D:\\Output"]
    /// </summary>
    public List<string> Xargs { get; set; } = new();

    // ── 高级可选参数 ──────────────────────────────────────

    /// <summary>
    /// Schema 收集器名称 (--schemaCollector / -s)
    /// 默认: "default"
    /// </summary>
    public string SchemaCollector { get; set; } = "default";

    /// <summary>
    /// Pipeline 名称 (--pipeline / -p)
    /// 默认: "default"
    /// </summary>
    public string Pipeline { get; set; } = "default";

    /// <summary>
    /// 只输出指定表 (--outputTable / -o, 可多个)
    /// </summary>
    public List<string> OutputTables { get; set; } = new();

    /// <summary>
    /// 包含标签过滤 (--includeTag / -i, 可多个)
    /// </summary>
    public List<string> IncludeTags { get; set; } = new();

    /// <summary>
    /// 排除标签过滤 (--excludeTag / -e, 可多个)
    /// </summary>
    public List<string> ExcludeTags { get; set; } = new();

    /// <summary>
    /// 字段变体 (--variant, 格式: key=value, 可多个)
    /// </summary>
    public List<string> Variants { get; set; } = new();

    /// <summary>
    /// 时区 (--timeZone)
    /// 例: "Asia/Shanghai"
    /// </summary>
    public string TimeZone { get; set; }

    /// <summary>
    /// 自定义模板目录列表 (--customTemplateDir, 可多个)
    /// </summary>
    public List<string> CustomTemplateDirs { get; set; } = new();

    /// <summary>
    /// 验证失败时作为错误退出 (--validationFailAsError)
    /// </summary>
    public bool ValidationFailAsError { get; set; }

    /// <summary>
    /// NLog 配置文件路径 (--logConfig / -l)
    /// 默认: "nlog.xml"
    /// </summary>
    public string LogConfig { get; set; } = "nlog.xml";

    /// <summary>
    /// 监听目录列表，变更时自动重新生成 (--watchDir / -w, 可多个)
    /// </summary>
    public List<string> WatchDirs { get; set; } = new();

    /// <summary>
    /// 无 dataTarget 时强制加载表数据 (--forceLoadTableDatas / -f)
    /// </summary>
    public bool ForceLoadTableDatas { get; set; }

    /// <summary>
    /// 详细输出模式 (--verbose / -v)
    /// </summary>
    public bool Verbose { get; set; }

    /// <summary>
    /// 验证所有必填项是否有效
    /// </summary>
    public bool IsValid()
    {
        return !string.IsNullOrWhiteSpace(LubanPath) &&
               !string.IsNullOrWhiteSpace(ConfFile) &&
               !string.IsNullOrWhiteSpace(Target);
    }
}
```

### ExportResult (导表结果)

```csharp
public class ExportResult
{
    /// <summary>
    /// 是否成功导表
    /// </summary>
    public bool Success { get; set; }
    
    /// <summary>
    /// 如果失败，记录错误信息
    /// </summary>
    public string ErrorMessage { get; set; }
    
    /// <summary>
    /// 完整的标准输出（stdout）
    /// </summary>
    public string StandardOutput { get; set; }
    
    /// <summary>
    /// 完整的标准错误（stderr）
    /// </summary>
    public string StandardError { get; set; }
    
    /// <summary>
    /// Luban 进程的退出码
    /// 0 = 成功，其他 = 失败
    /// </summary>
    public int ExitCode { get; set; }
    
    /// <summary>
    /// 导表耗时
    /// </summary>
    public TimeSpan Duration { get; set; }
}
```

```csharp
public class LogEntry
{

    /// 日志级别
```

## 总结

### 设计原则

1. **用户优先** - 每个功能都来自实际用户需求
2. **循序渐进** - MVP 先做最核心的事，后续逐步优化
3. **反馈优先** - 日志和进度反馈是最重要的体验
4. **开源友好** - 代码清晰、文档完善、易于扩展

### MVP 核心亮点

🔥 **实时日志反馈**: 用户能看到导表的每一步  
⚡ **零学习成本**: 完全的图形界面，不需要学 CLI  
💾 **智能配置保存**: 第一次配置后，后续一键导表  
✅ **清晰的反馈**: 成功和失败都有明确的提示
