## 产品概述

### 定位

**Luban-GUI** 是一个 Windows 桌面应用，为 Luban 配置表工作流提供完整的项目管理和可视化界面。

它**不是**简单的 Luban CLI 参数拼装器，而是一个拥有自身业务逻辑的**配置表管理工具**：

- 在 GUI 内部维护一套完整的 Luban 工作区（`luban.conf`、元数据表、Excel 数据文件）
- 提供创建表格、枚举、Bean 的快捷操作，自动生成符合 Luban 规范的 xlsx 文件
- 提供表格数据预览，并支持双击列快速定位到 Excel 文件的对应字段
- 让用户只需关心"导出什么类型、输出到哪里"，底层 Luban CLI 参数由 GUI 自动组装

### 解决的核心问题

| 问题 | 原因 | 解决方案 |
|-----|------|--------|
| 工作区分散难以管理 | Excel 文件、luban.conf、元数据表各自独立维护 | GUI 统一管理项目工作区 |
| 创建新表格繁琐 | 需要手动创建 xlsx 并维护 `__tables__.xlsx` | 快捷操作一键生成符合规范的表格 |
| 导出配置参数复杂 | Luban CLI 参数众多且互相关联 | 用户只填类型和路径，GUI 自动组装参数 |
| 数据需要打开 Excel 才能查看 | Luban 本身不提供预览 | GUI 内置表格数据预览 |
| 定位字段需要手动搜索 | Excel 文件列多时查找字段耗时 | 双击列名直接定位到对应 Excel 位置 |
| 执行过程不透明 | 不知道导表进度 | 实时日志窗口 |

### 目标用户

- **策划**: 需要快速维护和导出配置表，无需学习命令行或 Luban CLI 参数
- **程序员**: 集成流程中需要导表，并可从日志快速定位 Luban 报错
- **测试**: 验证表格导出结果，需要清晰的成功/失败提示

## 需求分析

### 用户故事

#### 故事1: 管理配置表工作区
**As a** 策划  
**I want to** 在一个地方管理所有配置表文件  
**So that** 我不需要手动维护 `luban.conf` 和元数据表，也不需要了解 Luban 的目录结构约定

**验收条件**:
- 可以新建/打开/切换项目
- 新建项目后工作区文件自动生成
- 在文件管理器中打开项目文件夹，可直接拖入 Excel 文件并在 GUI 中刷新识别

#### 故事2: 快速创建新表格
**As a** 策划  
**I want to** 在 GUI 中点几下就创建一张新的配置表  
**So that** 我不用手动创建 xlsx 文件，也不用手动编辑 `__tables__.xlsx`

**验收条件**:
- 弹出对话框，输入表名和字段列表（字段名 + 类型）
- GUI 自动在 `Datas/` 下生成符合 Luban 规范的 xlsx（含 `##` 元数据行）
- GUI 自动将新表注册到 `__tables__.xlsx`

#### 故事3: 预览表格数据并快速定位字段
**As a** 策划  
**I want to** 在 GUI 中查看表格数据，并快速跳转到对应的 Excel 字段  
**So that** 我不需要打开 Excel 并手动查找字段位置

**验收条件**:
- 单击左侧表格列表，右侧面板显示该表数据预览
- 双击列标题（字段名），自动打开对应 Excel 文件并定位到该列

#### 故事4: 一键导表（零参数配置）
**As a** 策划  
**I want to** 只需填写导出类型和输出路径，就能完成导表  
**So that** 我不需要了解 `--conf`、`--codeTarget`、`--xargs` 等 CLI 参数

**验收条件**:
- 导出配置窗口只有：数据类型、数据输出路径、代码类型、代码输出路径、导出目标
- 点击「全量导表」后 GUI 自动组装 Luban 参数并执行
- 实时日志窗口显示执行过程
- 成功和失败有清晰的视觉反馈

#### 故事5: 快速定位导表错误
**As a** 程序员  
**I want to** 看到详细的导表日志  
**So that** 我能快速找出哪张表有问题、哪一行出错

**验收条件**:
- 日志窗口显示所有 stdout 和 stderr，含 Luban 原始输出
- 错误信息能复制粘贴，方便分享
- 日志级别着色，错误一眼可见

## 功能设计

### MVP 版本 (v1.0) - 核心功能

#### ✅ F1: 项目管理

**描述**: 用户可以在 GUI 中创建、打开和切换 Luban 项目工作区。

**项目工作区结构**（由 GUI 自动维护）:
```
<WorkspaceRoot>/<ProjectName>/
├── projectConfig.json        # GUI 专属导出配置
├── luban.conf                # Luban 全局配置（自动生成和维护）
├── Defines/                  # 自定义类型定义（用户可选填写）
└── Datas/
    ├── __tables__.xlsx        # 表格元数据（GUI 维护，用户不需手动编辑）
    ├── __beans__.xlsx         # Bean 元数据（GUI 维护）
    ├── __enums__.xlsx         # 枚举元数据（GUI 维护）
    └── *.xlsx                 # 用户数据表
```

**操作**:
- **新建项目**: 输入名称和工作区路径 → GUI 自动生成完整目录和初始文件
- **打开项目**: 选择已有项目目录 → 读取 `projectConfig.json`
- **切换项目**: 下拉框切换 → 表格列表和内容区刷新
- **打开文件夹**: 在系统文件管理器中打开当前项目目录

---

#### ✅ F2: 快捷操作（表格/枚举/Bean 创建）

**描述**: 提供创建新表格、枚举类型、Bean 类型的快捷入口，自动按 Luban 规范生成文件。

**新建表格**:
- 输入表名和字段列表（字段名 + 类型）
- GUI 在 `Datas/` 下生成对应 xlsx，包含正确的 `##` 元数据行
- GUI 自动更新 `__tables__.xlsx`，将新表注册进去

**新建枚举**:
- 输入枚举名和枚举值列表
- GUI 自动更新 `__enums__.xlsx`

**新建 Bean**:
- 输入 Bean 名和字段列表（字段名 + 类型）
- GUI 自动更新 `__beans__.xlsx`

**导入已有文件**:
- 文件选择对话框，或直接拖入文件（在项目文件夹中放好后刷新识别）
- GUI 弹出提示，询问是否注册到 `__tables__.xlsx`

---

#### ✅ F3: 表格数据预览

**描述**: 在 GUI 右侧面板中预览 xlsx 数据内容，无需打开 Excel。

**特性**:
- 读取 xlsx 文件，跳过 `##` 开头的 Luban 元数据行，展示实际数据
- 虚拟化渲染，支持 1000+ 行不卡顿
- 点击列标题按该列排序
- 底部显示总行数
- 读取失败时显示具体错误原因

---

#### ✅ F4: 双击列快速定位

**描述**: 在表格内容面板中双击列标题（字段名），快速打开对应 Excel 文件并定位到该列。

**行为**:
- **Windows + Excel**: 通过 COM 自动化打开文件并定位到对应列
- **其他环境**: 退化为直接打开文件（使用系统默认程序）

---

#### ✅ F5: 简化导出配置

**描述**: 用户只需填写导出类型和输出路径，无需接触 Luban CLI 参数。

**用户填写内容**:
```
数据导出:
  • 导出类型（bin / json / lua 等）
  • 输出路径

代码导出（可选）:
  • 是否生成代码
  • 代码类型（cs-bin / typescript-json / java-bin 等）
  • 输出路径
  • 顶层命名空间（如 cfg）

导出目标:
  • 目标（从 luban.conf 动态读取，如 all / client / server）
```

GUI 根据上述配置自动组装完整的 Luban CLI 参数，**用户不需要了解任何命令行参数**。

**保存位置**: 项目目录下的 `projectConfig.json`

---

#### ✅ F6: 一键导表

**描述**: 用户点击「全量导表」，GUI 自动完成验证、参数组装、进程调用的全部流程。

**流程**:
```
1. 用户点击【全量导表】
   ↓
2. GUI 验证 projectConfig.json（输出路径非空、类型有效）
   ↓
3. GUI 确保输出目录存在（自动创建）
   ↓
4. GUI 组装 Luban CLI 参数（用户不可见）
   ↓
5. 启动内置 Luban CLI 子进程
   ↓
6. 异步读取 stdout/stderr，实时显示日志
   ↓
7. 进程完成，统计耗时
   ↓
8. 显示 ✓ 成功 / ✗ 失败，更新状态栏
```

**核心要求**:
- 🔥 **实时日志**: 异步读取进程输出，不阻塞 UI
- 🔄 **支持取消**: 随时点击【取消】中止导表
- ⚠️ **异常处理**: 进程出错时清晰显示 stderr
- ⏱️ **显示耗时**: 记录导表花费的时间

---

#### ✅ F7: 实时日志窗口

**描述**: 独立浮动窗口，实时展示 Luban 进程的执行日志。

**特性**:
- 逐行显示 stdout 和 stderr，前缀区分来源
- 按日志级别着色（INFO 灰 / SUCCESS 绿 / ERROR 红 / WARN 黄）
- 自动滚动，手动向上滚动时暂停并出现"↓ 跳到最新"按钮
- 支持全选和复制，便于分享错误信息
- 每次新导表启动前自动清空
- 虚拟化渲染，支持 10000+ 条日志不卡顿

## 用户交互流

### 核心场景1: 首次使用

```
1. 用户启动应用
   ↓
2. 弹出引导提示："请新建或打开一个项目"
   ↓
3. 用户点击【新建项目】，输入项目名称，选择工作区路径
   ↓
4. GUI 自动生成工作区（luban.conf + Datas/ + 三个元数据 xlsx）
   ↓
5. 用户点击【新建表格】，填写表名和字段
   ↓
6. GUI 在 Datas/ 下生成 xlsx，更新 __tables__.xlsx
   ↓
7. 用户在 Excel 中填写数据，保存
   ↓
8. 用户回到 GUI，点击【导出配置】，填写输出类型和路径
   ↓
9. 用户点击【全量导表】
   → 日志窗口显示：
     [INFO] 验证配置...
     [INFO] 启动 Luban 进程...
     [OUT] Processing...
     [SUCCESS] 导表完成 (耗时 2.34s)
   → 显示 ✓ 成功
   ↓
10. 下次启动时配置和项目记录被保留，直接点击【全量导表】即可
```

### 核心场景2: 日常导表（熟悉后）

```
1. 用户启动应用（上次项目自动加载）
   ↓
2. 查看左侧表格列表，确认需要更新的表
   ↓
3. 单击表格，右侧预览数据，确认内容
   ↓
4. 若需要定位某字段，双击列标题 → Excel 自动打开并定位
   ↓
5. 修改 Excel，保存
   ↓
6. 回到 GUI，点击【全量导表】
   ↓
7. 查看日志，确认成功
```

### 核心场景3: 导表失败排查

```
1. 用户点击【全量导表】
   ↓
2. Luban 因表格错误失败
   → 日志显示：
     [OUT] Processing...
     [ERR] Invalid column type at Sheet!A1
     [ERR] Exit code: 1
   ↓
3. UI 显示 ✗ 失败
   ↓
4. 用户复制日志错误信息
   ↓
5. 双击对应表格的问题列标题 → Excel 自动定位到问题字段
   ↓
6. 修复后重新点击【全量导表】
```

### 核心场景4: 用户取消

```
1. 用户点击【全量导表】，导表进行中
   ↓
2. 用户改主意，点击【取消】
   → Luban 进程被中止
   → 日志显示 [WARN] 用户中止
   ↓
3. UI 返回就绪状态
```

## 数据模型

### ProjectConfig（用户导出配置）

用户通过导出配置窗口填写，保存在项目目录下的 `projectConfig.json`。

```csharp
public class ProjectConfig
{
    /// <summary>
    /// 数据导出配置
    /// </summary>
    public DataOutputConfig DataOutput { get; set; } = new();

    /// <summary>
    /// 代码导出配置（可选）
    /// </summary>
    public CodeOutputConfig CodeOutput { get; set; } = new();

    /// <summary>
    /// 导出目标，对应 luban.conf 中定义的 targets，如 "all"、"client"、"server"
    /// </summary>
    public string Target { get; set; } = "all";
}

public class DataOutputConfig
{
    /// <summary>
    /// 数据序列化格式，如 "bin"、"json"、"lua"
    /// </summary>
    public string Type { get; set; } = "bin";

    /// <summary>
    /// 导出数据文件的目标目录
    /// </summary>
    public string OutputPath { get; set; } = string.Empty;
}

public class CodeOutputConfig
{
    /// <summary>
    /// 是否生成代码文件
    /// </summary>
    public bool Enabled { get; set; } = false;

    /// <summary>
    /// 代码目标，如 "cs-bin"、"typescript-json"、"java-bin"
    /// </summary>
    public string Type { get; set; } = "cs-bin";

    /// <summary>
    /// 生成代码文件的目标目录
    /// </summary>
    public string OutputPath { get; set; } = string.Empty;

    /// <summary>
    /// 生成代码的顶层命名空间，如 "cfg"
    /// </summary>
    public string TopModule { get; set; } = "cfg";
}
```

**JSON 示例**:
```json
{
  "dataOutput": {
    "type": "bin",
    "outputPath": "D:/Game/Assets/Config"
  },
  "codeOutput": {
    "enabled": true,
    "type": "cs-bin",
    "outputPath": "D:/Game/Scripts/Gen",
    "topModule": "cfg"
  },
  "target": "all"
}
```

### ProjectInfo（项目元信息）

保存在全局 `appConfig.json` 中，用于项目列表管理。

```csharp
public class ProjectInfo
{
    /// <summary>
    /// 项目名称（同时也是工作区目录名）
    /// </summary>
    public string Name { get; set; } = string.Empty;

    /// <summary>
    /// 工作区根目录（包含项目子目录的父目录）
    /// </summary>
    public string WorkspaceRoot { get; set; } = string.Empty;

    /// <summary>
    /// 完整的项目目录路径 = WorkspaceRoot / Name
    /// </summary>
    public string ProjectPath => Path.Combine(WorkspaceRoot, Name);

    /// <summary>
    /// 最后打开时间（用于排序）
    /// </summary>
    public DateTime LastOpenedAt { get; set; }
}
```

### LogEntry（日志条目）

public class LogEntry

## 总结

### 设计原则

1. **以项目为中心** - GUI 维护完整的 Luban 工作区，而非仅传参数给 CLI
2. **用户优先** - 用户只关心"导什么、输出到哪"，不需要了解 Luban 内部机制
3. **快捷操作** - 常见操作（创建表格/枚举/Bean）一键完成，自动符合 Luban 规范
4. **反馈优先** - 日志、预览、状态反馈贯穿整个使用流程
5. **循序渐进** - MVP 先实现核心功能，后续根据实际使用反馈迭代

### MVP 核心亮点

🗂️ **项目工作区管理**: GUI 自动维护 `luban.conf` 和元数据表  
⚡ **快捷创建**: 新建表格/枚举/Bean，自动生成符合规范的文件  
👁️ **数据预览**: 无需打开 Excel 即可查看表格内容  
🎯 **双击定位**: 双击列名直接跳转到 Excel 对应字段  
🔥 **零参数导表**: 只填类型和路径，一键完成  
📋 **实时日志**: 导表全过程透明可见
