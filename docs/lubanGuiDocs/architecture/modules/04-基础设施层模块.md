# 基础设施层模块设计

**所属层次**: 基础设施层 (Infrastructure)  
**版本**: 2.0

---

## 职责

提供持久化、日志等横切关注点，不包含业务逻辑。基础设施层是最底层，不依赖其他层。

---

## AppConfigManager

**职责**: 管理全局应用配置（已注册项目列表、上次打开的项目、主题等）

**存储位置**:
```
Windows: %LOCALAPPDATA%\LubanGui\appConfig.json
```

**配置内容示例**:
```json
{
  "lastOpenedProject": "MyGame",
  "registeredProjects": [
    {
      "name": "MyGame",
      "workspacePath": "D:/Projects/MyGame"
    }
  ],
  "theme": "Dark"
}
```

**主要操作**:

| 操作 | 说明 |
|------|------|
| `LoadAsync()` | 从磁盘读取全局配置，文件不存在时返回默认值 |
| `SaveAsync(AppConfig)` | 将全局配置写回磁盘 |
| `AddProject(ProjectInfo)` | 追加一条项目记录并持久化 |
| `RemoveProject(string name)` | 删除一条项目记录并持久化 |

## ProjectConfigManager

**职责**: 管理单个项目的 `projectConfig.json` 读写

**存储位置**:
```
<WorkspacePath>/<ProjectName>/projectConfig.json
```

**读写实现**:
```csharp
public async Task<ProjectConfig> LoadAsync(string projectDir)
{
    var path = Path.Combine(projectDir, "projectConfig.json");
    if (!File.Exists(path))
        return new ProjectConfig();
    using var stream = File.OpenRead(path);
    return await JsonSerializer.DeserializeAsync<ProjectConfig>(stream);
}

public async Task SaveAsync(string projectDir, ProjectConfig config)
{
    var path = Path.Combine(projectDir, "projectConfig.json");
    using var stream = File.Create(path);
    await JsonSerializer.SerializeAsync(stream, config, _jsonOptions);
}
```

**ProjectConfig 结构**（对应 `projectConfig.json`）:
```json
{
  "projectName": "MyGame",
  "target": "all",
  "dataOutput": {
    "type": "bin",
    "outputPath": "D:/Game/Assets/Config"
  },
  "codeOutput": {
    "enabled": true,
    "type": "cs-bin",
    "outputPath": "D:/Game/Scripts/Gen",
    "topModule": "cfg"
  }
}
```

## Logger

**职责**: 统一日志管理，封装 Serilog 初始化配置

**初始化配置**:
```csharp
Log.Logger = new LoggerConfiguration()
    .MinimumLevel.Information()
    .WriteTo.Console()
    .WriteTo.File("logs/luban-gui-.txt", rollingInterval: RollingInterval.Day)
    .CreateLogger();
```

**日志输出位置**:

| Sink | 位置 |
|------|------|
| Console | 标准输出（开发调试用） |
| File | `logs/luban-gui-<日期>.txt`，按天滚动 |

**日志级别约定**:

| 级别 | 使用场景 |
|------|----------|
| `Information` | 正常操作流程（项目创建、导出开始/完成） |
| `Warning` | 可恢复的异常情况（配置字段缺失，使用默认值） |
| `Error` | 操作失败（进程异常退出、文件 IO 失败） |
| `Debug` | 仅开发环境输出的诊断信息 |

---

## 层间约定

- 基础设施层不依赖任何其他层，是整个依赖图的底层。
- 所有文件 IO 操作必须是异步方法（`Task` / `Task<T>`）。
- `AppConfigManager` 和 `ProjectConfigManager` 均以单例形式注册到 DI 容器。
- Logger 在应用启动时初始化一次，通过 `ILogger` 接口注入使用。
