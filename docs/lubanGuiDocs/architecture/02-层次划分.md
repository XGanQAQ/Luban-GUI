## 五层架构概览

| 层次 | 职责 | 主要组件 |
|------|------|----------|
| **表现层** | UI 显示与用户交互，路由命令到 ViewModel | Views、ViewModels |
| **业务逻辑层** | 实现核心业务流程，协调各服务 | ProjectManager、SchemaService、ExportService、TablePreviewService |
| **工具层** | 封装底层 IO 和进程操作，供 Service 调用 | LubanExecutor、LubanCommandBuilder、ExcelWriter、FileOpenService |
| **基础设施层** | 持久化配置，提供日志等横切关注点 | AppConfigManager、ProjectConfigManager、Logger |
| **Luban 源适配层** | 隔离 GUI 与 Luban 源码，封装对 Luban 内部数据结构的读取 | LubanSchemaReader、LubanConfAdapter、LubanTypeMapper |

层间只向下依赖；上层通过接口调用下层，不直接访问下下层。**Luban 源适配层是 GUI 与 `lubanSrc\` 之间的唯一接触点**，上方各层不得直接引用 Luban 源码中的类型。

## 表现层 (Avalonia UI)

**职责**:
- 显示项目列表，支持切换
- 显示表格列表，支持快速操作（新建/导入）
- 预览表格数据，支持双击列定位到 Excel 文件
- 提供简化的导出配置窗口
- 路由命令到 ViewModel

**关键 ViewModel**:

| ViewModel | 职责 |
|-----------|------|
| `MainWindowViewModel` | 统筹项目切换、表格列表、表格预览、导表状态 |
| `TableListViewModel` | 管理左侧表格列表，响应快捷操作命令 |
| `TablePreviewViewModel` | 管理右侧表格数据预览，响应双击定位命令 |
| `ExportConfigViewModel` | 管理导出配置（数据类型/路径、代码类型/路径） |
| `LogWindowViewModel` | 管理日志浮动窗口 |

详细 UI 设计见 [../04-前端界面文档.md](../04-前端界面文档.md)，各 ViewModel 内部设计见 [modules/01-表现层模块.md](./modules/01-表现层模块.md)。

## 业务逻辑层 (Service)

**职责**: 实现核心业务流程，协调工具层完成复杂操作。

**组件**:

| 服务 | 职责 |
|------|------|
| `ProjectManager` | 管理多个项目的创建、切换、加载，初始化工作区 |
| `SchemaService` | 维护 Luban 元数据表，提供创建新表/枚举/Bean 操作 |
| `ExportService` | 根据 `ProjectConfig` 编排完整的导表流程 |
| `TablePreviewService` | 读取 Excel 数据文件，为右侧面板提供预览数据 |

详细接口和流程设计见 [modules/02-业务逻辑层模块.md](./modules/02-业务逻辑层模块.md)。

## 工具层 (Executor / IO)

**职责**: 封装底层 IO 和进程操作，不包含业务逻辑。

**组件**:

| 工具 | 职责 |
|------|------|
| `LubanExecutor` | 启动 Luban CLI 子进程，捕获 stdout/stderr 输出流 |
| `LubanCommandBuilder` | 将 `ProjectConfig` 转换为完整的 Luban CLI 参数（内部使用） |
| `ExcelWriter` | 按 Luban 规范生成和写入 xlsx 文件（使用 ClosedXML） |
| `FileOpenService` | 打开 Excel 文件并可选地定位到指定字段 |

详细实现设计见 [modules/03-工具层模块.md](./modules/03-工具层模块.md)。

**职责**: 提供持久化、日志等横切关注点，不包含业务逻辑。

**组件**:

## Luban 源适配层 (LubanAdapter)

**职责**: 作为 GUI 与 `lubanSrc\` 源码之间的**唯一隔离边界**。GUI 各层所需的 Luban 内部数据（Schema 定义、配置格式、类型系统）全部通过此层的接口获取，上层不直接引用任何 `lubanSrc\` 中的类型。

**存在意义**:
- Luban 源码内部 API 随版本迭代可能变化；有此层后，适配改动只需在一处修改，不会扩散到整个 GUI 代码库。
- 接口语义以 GUI 需求为中心，屏蔽 Luban 内部复杂度（如多态 Bean 树、加载器链等）。
- 可通过 Mock 此层接口，在不依赖 Luban 源码的情况下独立测试上层逻辑。

**组件**:

| 组件 | 职责 |
|------|------|
| `LubanSchemaReader` | 从 `__tables__.xlsx`、`__enums__.xlsx`、`__beans__.xlsx` 读取 Schema 定义，返回 GUI 友好的 DTO |
| `LubanConfAdapter` | 解析和生成 `luban.conf`，屏蔽 Luban 配置格式细节 |
| `LubanTypeMapper` | 将 Luban 内置类型（`int`、`string`、`bool` 等）映射为 GUI 可显示的类型描述 |

详细接口设计见 [modules/05-Luban源适配层模块.md](./modules/05-Luban源适配层模块.md)。

### 方案1: 线程安全的 UI 更新

**问题**: LubanExecutor 在后台线程读取进程输出，如何安全地更新 UI？

**方案**: 使用 `IProgress<T>`

```csharp
// 后台线程中
logProgress?.Report(logLine);  // 线程安全，自动切换到 UI 线程

// ViewModel 中
var progress = new Progress<string>(line =>
{
    // 自动在 UI 线程执行
    LogItems.Add(line);
});
```

### 方案2: 实时日志显示

**问题**: 如果等到进程完成再显示日志，用户要等很久

**方案**: 逐行实时推送

```
进程输出 → StreamReader → 逐行读 → IProgress.Report() → UI 更新
     (异步)         (逐行)         (实时)          (自动线程切换)
```

**性能优化**:
- 使用虚拟化列表（ListBox.VirtualizationMode = Item）
- 限制日志条数（保留最近 10000 行）
- 异步写入磁盘

### 方案3: 进程间的输出捕获

**问题**: Luban 输出到 stdout/stderr，我们如何获取？

**方案**: 重定向并异步读取

```
Luban 进程
  ├─ stdout → RedirectStandardOutput=true → stream → ReadLineAsync()
  └─ stderr → RedirectStandardError=true → stream → ReadLineAsync()
```

**关键设置**:
```csharp
var psi = new ProcessStartInfo
{
    RedirectStandardOutput = true,      // ✓ 重定向 stdout
    RedirectStandardError = true,       // ✓ 重定向 stderr
    UseShellExecute = false,            // ✓ 不通过 cmd.exe
    CreateNoWindow = true,              // ✓ 不创建窗口
    StandardOutputEncoding = Encoding.UTF8  // ✓ UTF-8 编码
};
```

### 方案4: 避免进程通信死锁

**问题**: 进程缓冲区有限，如果只读一个流，当缓冲区满时会死锁

**解决**: 同时读取两个流

```csharp
// ✗ 死锁：先读完 stdout，再读 stderr，缓冲区满时卡住
string out = process.StandardOutput.ReadToEnd();
string err = process.StandardError.ReadToEnd();

// ✓ 正确：同时读两个流
Task<string> outTask = ReadStreamAsync(process.StandardOutput);
Task<string> errTask = ReadStreamAsync(process.StandardError);
await Task.WhenAll(outTask, errTask);
```

<ItemGroup>

    <PackageReference Include="Serilog.Sinks.File" Version="5.0.0" />

    <!-- 依赖注入（内置，显式声明） -->

</ItemGroup>

| 包 | 用途 |
|----|------|
| `Avalonia` | 跨平台 UI 框架 |
| `CommunityToolkit.Mvvm` | MVVM 基础设施（ObservableObject、RelayCommand） |

| `System.Text.Json` | 配置 JSON 读写（内置，无需额外引用） |
| `System.Diagnostics.Process` | 启动 Luban CLI 子进程（内置） |

### 项目结构

```
LubanGui/
├── LubanGui.csproj
├── App.axaml                       # 应用入口
├── Assets/                         # 图标、资源
├── Views/
│   ├── MainWindow.axaml            # 主窗口
│   ├── ExportConfigWindow.axaml    # 导出配置窗口（简化）
│   └── LogWindow.axaml             # 日志浮动窗口
├── ViewModels/
│   ├── MainWindowViewModel.cs
│   ├── TableListViewModel.cs
│   ├── TablePreviewViewModel.cs
│   ├── ExportConfigViewModel.cs
│   └── LogWindowViewModel.cs
├── Models/
│   ├── ProjectConfig.cs            # 用户可见的导出配置
│   ├── ProjectInfo.cs              # 项目元信息
│   ├── TableMeta.cs                # 表格元数据
│   ├── TablePreviewData.cs         # 表格预览数据
│   └── ExportResult.cs
├── Services/
│   ├── ProjectManager.cs           # 项目管理
│   ├── SchemaService.cs            # 表/枚举/Bean 创建
│   ├── ExportService.cs            # 导表流程
│   ├── TablePreviewService.cs      # 表格数据预览
│   └── Luban/
│       ├── LubanExecutor.cs        # 进程管理 + 输出捕获
│       └── LubanCommandBuilder.cs  # 参数构建（内部）
├── Infrastructure/
│   ├── AppConfigManager.cs         # 全局配置读写
│   ├── ProjectConfigManager.cs     # 项目配置读写
│   ├── ExcelWriter.cs              # xlsx 生成
│   └── FileOpenService.cs          # 打开文件并定位
└── Datas/                          # 内置 Luban 模板表格（随 GUI 分发）
    ├── __tables__.xlsx
    ├── __beans__.xlsx
    └── __enums__.xlsx
```

### 编译和运行

```bash
# 恢复依赖
dotnet restore

# 编译（同时构建内置 Luban CLI）
dotnet build .\LubanGui.slnx

# 运行
dotnet run --project .\LubanGui\LubanGui.csproj

# 发布
dotnet publish .\LubanGui\LubanGui.csproj -c Release -o .\publish
```
