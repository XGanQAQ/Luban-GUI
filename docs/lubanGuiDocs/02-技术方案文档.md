## 技术选型

### 技术栈

| 下沉层级 | 技术 | 版本 | 说明 |
|---------|------|------|------|
| **UI框架** | Avalonia | 11.x | 跨平台 XAML 框架，开源，C# 生态 |
| **编程语言** | C# | 10+ | 类型安全，LINQ，现代语言特性 |
| **配置管理** | System.Text.Json | 内置 | 轻量，无外部依赖 |
| **进程通信** | System.Diagnostics.Process | 内置 | 标准库，稳定可靠 |
| **日志** | Serilog | 3.x | 结构化日志，灵活配置，性能好 |
| **依赖注入** | Microsoft.Extensions.DependencyInjection | 8.0 | 标准库，Avalonia 原生支持 |

### 选型原因

#### 为什么选 Avalonia？

✅ **开源免费** - MIT 许可，无商业限制  
✅ **跨平台** - 同一套代码支持 Windows/Linux/macOS  
✅ **XAML 支持** - 熟悉 WPF 的开发者易上手  
✅ **完整 UI 库** - Dialog、ListBox 等常用控件齐全  
✅ **社区活跃** - 问题容易找到答案  
✅ **C# 生态** - 与 .NET 框架完美集成

#### 为什么不用 WPF？

❌ Windows only  
❌ 过时，官方不推荐新项目使用  
❌ 性能不如 Avalonia

#### 为什么不用 WinUI 3？

❌ 学习成本高，生态不成熟  
❌ 对个人开发者不友好

#### 为什么选 System.Text.Json？

✅ **内置库** - 不引入第三方依赖  
✅ **性能好** - 比 Newtonsoft.Json 快  
✅ **轻量** - MVP 阶段配置简单，足够用  
✅ **AOT 友好** - 支持 Native AOT 优化

#### 为什么选 Serilog？

✅ **结构化日志** - 便于后续分析  
✅ **灵活 Sink** - 支持文件、控制台、数据库等  
✅ **性能高** - 异步写入，不阻塞主线程  
✅ **社区成熟** - Nuget 下载量最高

#### ExportService

**职责**: 编排整个导表流程

**接口**:
```csharp
public interface IExportService
{
    Task<ExportResult> ExportAllAsync(
        ExportConfig config,
        IProgress<string> logProgress,
        CancellationToken cancellationToken = default);
    
    ValidationResult ValidateConfig(ExportConfig config);
}
```

**流程**:
```
1. 验证配置（LubanPath/ConfFile/Target 不为空，路径存在）
   ↓
2. 调用 LubanExecutor 执行导表
   ↓
3. 保存配置到本地（用户首次成功时）
   ↓
4. 返回结果
```

**异常处理**:
- ConfigInvalidException → 显示配置错误
- ProcessException → 显示 Luban 错误
- OperationCanceledException → 显示"已取消"
- 通用 Exception → 显示"未知错误"

#### LubanCommandBuilder

**职责**: 根据 ExportConfig 构建 Luban 命令行参数

**实现**:
```csharp
public class LubanCommandBuilder
{
    public string BuildExportArgs(ExportConfig config)
    {
        var args = new StringBuilder();

        // 必填参数
        args.Append($"--conf \"{config.ConfFile}\" ");
        args.Append($"--target {config.Target} ");

        // 可选: schema 收集器 / pipeline（有默认值，非默认时才传）
        if (!string.IsNullOrEmpty(config.SchemaCollector) && config.SchemaCollector != "default")
            args.Append($"--schemaCollector {config.SchemaCollector} ");
        if (!string.IsNullOrEmpty(config.Pipeline) && config.Pipeline != "default")
            args.Append($"--pipeline {config.Pipeline} ");

        // 可重复参数
        foreach (var ct in config.CodeTargets)
            args.Append($"--codeTarget {ct} ");
        foreach (var dt in config.DataTargets)
            args.Append($"--dataTarget {dt} ");
        foreach (var ot in config.OutputTables)
            args.Append($"--outputTable {ot} ");
        foreach (var tag in config.IncludeTags)
            args.Append($"--includeTag {tag} ");
        foreach (var tag in config.ExcludeTags)
            args.Append($"--excludeTag {tag} ");
        foreach (var v in config.Variants)
            args.Append($"--variant {v} ");
        foreach (var x in config.Xargs)
            args.Append($"--xargs {x} ");
        foreach (var dir in config.CustomTemplateDirs)
            args.Append($"--customTemplateDir \"{dir}\" ");
        foreach (var dir in config.WatchDirs)
            args.Append($"--watchDir \"{dir}\" ");

        // 单值可选参数
        if (!string.IsNullOrEmpty(config.TimeZone))
            args.Append($"--timeZone {config.TimeZone} ");
        if (!string.IsNullOrEmpty(config.LogConfig) && config.LogConfig != "nlog.xml")
            args.Append($"--logConfig \"{config.LogConfig}\" ");

        // 布尔开关
        if (config.ValidationFailAsError)
            args.Append("--validationFailAsError ");
        if (config.ForceLoadTableDatas)
            args.Append("--forceLoadTableDatas ");
        if (config.Verbose)
            args.Append("--verbose ");

        return args.ToString().TrimEnd();
    }
}
```

**参数映射**:

| 配置字段 | Luban CLI 参数 | 短选项 | 类型 | 说明 |
|---------|--------------|--------|------|------|
| `ConfFile` | `--conf` | — | string (必填) | Luban 配置文件路径 |
| `Target` | `--target` | `-t` | string (必填) | 生成目标名称 |
| `CodeTargets` | `--codeTarget` | `-c` | string[] | 代码输出目标，可多次 |
| `DataTargets` | `--dataTarget` | `-d` | string[] | 数据输出目标，可多次 |
| `Xargs` | `--xargs` | `-x` | string[] (key=val) | 扩展参数，可多次 |
| `SchemaCollector` | `--schemaCollector` | `-s` | string | Schema 收集器，默认 "default" |
| `Pipeline` | `--pipeline` | `-p` | string | Pipeline，默认 "default" |
| `OutputTables` | `--outputTable` | `-o` | string[] | 只输出指定表，可多次 |
| `IncludeTags` | `--includeTag` | `-i` | string[] | 包含标签，可多次 |
| `ExcludeTags` | `--excludeTag` | `-e` | string[] | 排除标签，可多次 |

#### ProcessManager

**职责**: 启动子进程，管理其生命周期，捕获输出

**关键特性**:
- 🔥 **异步非阻塞** - 使用 StreamReader.ReadLineAsync() 逐行读取
- 🌐 **并行捕获** - Task.WhenAll 同时读 stdout/stderr，避免死锁
- 📡 **实时反馈** - 通过 IProgress<string> 推送日志到 UI
- ⏹️ **取消支持** - 支持 CancellationToken 中断进程

**核心代码**:
```csharp
public async Task<ProcessResult> RunAsync(
    string exePath,
    string arguments,
    IProgress<string> logProgress,
    CancellationToken cancellationToken)
{
    var psi = new ProcessStartInfo
    {
        FileName = exePath,
        Arguments = arguments,
        RedirectStandardOutput = true,
        RedirectStandardError = true,
        UseShellExecute = false,
        CreateNoWindow = true,
        StandardOutputEncoding = Encoding.UTF8,
        StandardErrorEncoding = Encoding.UTF8
    };
    
    var process = Process.Start(psi);
    
    // 并行读取两个流，避免死锁
    var outTask = ReadStreamAsync(process.StandardOutput, "[OUT]", logProgress);
    var errTask = ReadStreamAsync(process.StandardError, "[ERR]", logProgress);
    
    await process.WaitForExitAsync(cancellationToken);
    await Task.WhenAll(outTask, errTask);
    
    return new ProcessResult { ... };
}

private async Task<string> ReadStreamAsync(
    StreamReader stream,
    string prefix,
    IProgress<string> logProgress)
{
    var sb = new StringBuilder();
    string line;
    while ((line = await stream.ReadLineAsync()) != null)
    {
        var logLine = $"{prefix} {line}";
        sb.AppendLine(logLine);
        logProgress?.Report(logLine);  // 实时推送到 UI
    }
    return sb.ToString();
}
```

**为什么这样设计**:

❓ 为什么异步读取？  
➜ 同步读取会阻塞 UI 线程，导致应用卡顿

❓ 为什么用 Task.WhenAll？  
➜ 避免死锁。如果进程缓冲区满，同步读一个流会导致另一个流永远读不到

❓ 为什么用 IProgress<T>？  
➜ 线程安全。自动处理跨线程 UI 更新，无需手动 Invoke

#### LubanExecutor

**职责**: 整合 CommandBuilder 和 ProcessManager，提供高层接口

```csharp
public interface ILubanExecutor
{
    Task<LubanResult> ExecuteExportAsync(
        ExportConfig config,
        IProgress<string> logProgress,
        CancellationToken cancellationToken);
}

public class LubanExecutor : ILubanExecutor
{
    public async Task<LubanResult> ExecuteExportAsync(...)
    {
        // 1. 构建参数
        var args = _commandBuilder.BuildExportArgs(config);
        logProgress?.Report($"[INFO] 命令: {args}");
        
        // 2. 启动进程
        var processResult = await _processManager.RunAsync(
            config.LubanPath, args, logProgress, cancellationToken);
        
        // 3. 返回结果
        return new LubanResult
        {
            Success = processResult.ExitCode == 0,
            StandardOutput = processResult.StandardOutput,
            StandardError = processResult.StandardError,
            ExitCode = processResult.ExitCode
        };
    }
}
```

**读写操作**:
```csharp
public async Task<ExportConfig> LoadConfigAsync()
{
    var path = GetConfigPath();
    if (!File.Exists(path))
        return new ExportConfig();  // 返回默认配置
    
    using var stream = File.OpenRead(path);
    return await JsonSerializer.DeserializeAsync<ExportConfig>(stream);
}

public async Task SaveConfigAsync(ExportConfig config)
{
    var path = GetConfigPath();
    Directory.CreateDirectory(Path.GetDirectoryName(path));
    
    using var stream = File.Create(path);
    await JsonSerializer.SerializeAsync(stream, config);
}
```

## 关键技术方案

### 方案1: 线程安全的 UI 更新

**问题**: ProcessManager 在后台线程读取进程输出，如何安全地更新 UI？

**方案**: 使用 IProgress<T>

```csharp
// 后台线程中
logProgress?.Report(logLine);  // 线程安全，自动切换到 UI 线程

// ViewModel 中
var progress = new Progress<string>(line =>
{
    // 自动在 UI 线程执行
    LogItems.Add(line);
});
```

**优势**:
- ✅ 标准库，无需第三方依赖
- ✅ 线程安全，自动处理线程同步
- ✅ 支持取消令牌

### 方案2: 实时日志显示

**问题**: 如果等到进程完成再显示日志，用户要等很久

**方案**: 逐行实时推送

```
进程输出 → StreamReader → 逐行读 → IProgress.Report() → UI 更新
     (异步)         (逐行)         (实时)          (自动线程切换)
```

**性能优化**:
- 使用虚拟化列表（ListBox.VirtualizationMode = Item）
- 限制日志条数（保留最近 10000 行）
- 异步写入磁盘

### 方案3: 进程间的输出捕获

**问题**: Luban 输出到 stdout/stderr，我们如何获取？

**方案**: 重定向并异步读取

```
Luban 进程
  ├─ stdout → RedirectStandardOutput=true → stream → ReadLineAsync()
  └─ stderr → RedirectStandardError=true → stream → ReadLineAsync()
```

**关键设置**:
```csharp
var psi = new ProcessStartInfo
{
    RedirectStandardOutput = true,      // ✓ 重定向 stdout
    RedirectStandardError = true,       // ✓ 重定向 stderr
    UseShellExecute = false,            // ✓ 不通过 cmd.exe
    CreateNoWindow = true,              // ✓ 不创建窗口
    StandardOutputEncoding = Encoding.UTF8  // ✓ UTF-8 编码
};
```

### 方案4: 避免进程通信死锁

**问题**: 进程缓冲区有限，如果只读一个流，当缓冲区满时会死锁

**解决**: 同时读取两个流

```csharp
// ✗ 死锁：先读完 stdout，再读 stderr，缓冲区满时卡住
string out = process.StandardOutput.ReadToEnd();
string err = process.StandardError.ReadToEnd();

// ✓ 正确：同时读两个流
Task<string> outTask = ReadStreamAsync(process.StandardOutput);
Task<string> errTask = ReadStreamAsync(process.StandardError);
await Task.WhenAll(outTask, errTask);
```
