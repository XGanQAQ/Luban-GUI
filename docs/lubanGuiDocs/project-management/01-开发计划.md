## 二、第一阶段：MVP 开发（v1.0）

### 阶段目标

实现完整的项目管理和基础导表功能。用户可以在 GUI 中创建/打开项目，新建表格/枚举/Bean，预览数据，并一键完成导表。

### 完成标准

- [ ] 项目管理（新建/打开/切换项目）
- [ ] 快捷操作（新建表格/枚举/Bean，导入文件）
- [ ] 表格预览（读取 xlsx 数据显示）
- [ ] 双击列定位（打开对应 Excel 文件）
- [ ] 简化导出配置（仅填写类型和路径）
- [ ] 一键导表（调用内置 Luban CLI）
- [ ] 实时日志

---

### Week 1：项目框架搭建

**目标**：建立可运行的项目框架，确定核心架构。

**任务**：

1. **项目初始化**
   - 使用 Avalonia 模板创建项目
   - 配置 `.csproj`，添加必需的 NuGet 包（含 ClosedXML）
   - 验证项目编译和运行

2. **依赖注入配置**
   - 在 `App.axaml.cs` 中配置 DI 容器
   - 注册所有 Service、ViewModel、View
   - 验证依赖注入能正常工作

3. **日志系统初始化**
   - 配置 Serilog，输出到控制台和文件

4. **UI 布局基础**
   - 主窗口：菜单栏 + 项目切换栏 + 快捷工具栏 + 左右分栏 + 状态栏
   - 左侧表格列表面板（含过滤框）
   - 右侧表格内容面板（占位状态）

**验收标准**：
- [x] 项目编译通过，无编译错误
- [x] 应用能启动，显示主窗口布局
- [x] DI 容器能正常注入依赖

---

### Week 2：项目管理与工作区

**目标**：实现 ProjectManager，支持新建/打开/切换项目并初始化工作区。

**任务**：

1. **AppConfigManager 实现**
   - 存储位置：`%LOCALAPPDATA%\LubanGui\appConfig.json`
   - 保存已注册项目列表、上次打开的项目

2. **ProjectManager 实现**
   - `CreateProjectAsync`：创建目录，生成初始 `luban.conf`、`Datas/` 目录和三个元数据 xlsx
   - `OpenProjectAsync`：读取现有项目目录的 `projectConfig.json`
   - `SwitchProjectAsync`：切换当前项目，通知 UI 刷新

3. **ProjectConfigManager 实现**
   - 读写 `projectConfig.json`

4. **UI 绑定**
   - 项目切换栏下拉框绑定项目列表
   - 「新建项目」弹出输入对话框
   - 「打开项目文件夹」打开系统文件管理器

**验收标准**：
- [x] 能成功创建新项目，生成完整工作区文件结构
- [x] 能打开已有项目目录
- [x] 切换项目时 UI 正确刷新
- [x] 应用重启后能自动加载上次打开的项目

---

### Week 3：Schema 管理与表格预览

**目标**：实现 SchemaService 和 TablePreviewService，支持快捷操作和数据预览。

**任务**：

1. **ExcelWriter 实现（ClosedXML）**
   - 生成符合 Luban 规范的 xlsx（含 `##var`/`##type`/`##` 元数据行）
   - 向 `__tables__.xlsx`、`__enums__.xlsx`、`__beans__.xlsx` 追加/更新条目

2. **SchemaService 实现**
   - `CreateTableAsync`：生成 xlsx + 写入 `__tables__.xlsx`
   - `ImportTableAsync`：注册已有 xlsx
   - `CreateEnumAsync`、`CreateBeanAsync`

3. **TablePreviewService 实现（ClosedXML）**
   - 读取 xlsx，跳过 `##` 开头元数据行，提取数据行
   - `GetFieldLocation`：返回列号（用于双击定位）

4. **快捷操作 UI**
   - 「新建表格」对话框（表名 + 字段列表）
   - 「新建枚举」、「新建Bean」对话框
   - 「导入文件」文件选择对话框

5. **表格列表面板**
   - 绑定当前项目表格元数据
   - 单击加载右侧预览
   - 右键菜单

6. **表格内容面板**
   - DataGrid 显示预览数据（虚拟化）
   - 双击列标题调用 `FileOpenService`

7. **FileOpenService 实现**
   - `OpenFile`：`Process.Start` 打开文件
   - `OpenFileAtCell`（Windows 专属）：COM 自动化定位到指定列

**验收标准**：
- [x] 能通过 GUI 创建新表格，生成对应 xlsx 文件
- [x] 表格列表能正确显示当前项目所有表格
- [x] 单击表格在右侧显示数据预览
- [x] 双击列标题能打开对应 Excel 文件

---

### Week 4：导表流程与导出配置

**目标**：实现完整的一键导表流程，完成简化的导出配置窗口。

**任务**：

1. **LubanCommandBuilder 实现（内部）**
   - 从 `ProjectConfig` 映射到完整的 Luban CLI 参数
   - 正确组装 `--conf`、`--target`、`--codeTarget`、`--dataTarget`、`--xargs` 等

2. **LubanExecutor 实现**
   - 启动内置 Luban CLI 子进程
   - 异步并行捕获 stdout/stderr（避免死锁）
   - 通过 `IProgress<string>` 实时推送日志到 UI
   - 支持 `CancellationToken` 取消

3. **ExportService 实现**
   - 验证 `ProjectConfig`
   - 确保输出目录存在
   - 调用 `LubanExecutor`
   - 保存配置

4. **导出配置窗口**
   - 数据导出区：类型下拉 + 输出路径
   - 代码导出区：启用复选框 + 类型下拉 + 输出路径 + 命名空间
   - 导出目标下拉（从 `luban.conf` 动态读取）

5. **日志窗口**
   - 浮动独立窗口，实时日志显示
   - 颜色区分日志级别
   - 自动滚动 + 手动暂停

**验收标准**：
- [x] 导出配置窗口能正确保存到 `projectConfig.json`
- [x] 点击「全量导表」能调用 Luban CLI，显示实时日志
- [x] 导表成功后状态栏更新

## 三、第二阶段：功能优化（v1.1）

### 阶段目标

提升用户体验，支持多导出配置预设，初步支持增量导表。

### 功能清单

#### F1：多导出配置预设

同一项目下保存多套导出配置（如"out_client_bin"和"out_server_json"），快速切换。

- `ProjectConfig` 改为支持多预设列表
- 导出配置窗口顶部增加预设切换下拉框
- 支持新建、复制、删除、重命名预设

#### F2：增量导表基础

检测自上次导表后修改的 Excel 文件，只处理变化的表。

- 记录上次导表时间
- 比对 Excel 文件修改时间（`File.GetLastWriteTime`）
- 导表前提示"将处理 X 个修改的表"

#### F3：UI 优化

- 暗色模式支持（Avalonia FluentTheme Dark）
- 记住窗口大小和位置（写入 `appConfig.json`）
- 快捷键支持（`F5` 全量导表，`Ctrl+L` 日志窗口）

## 四、第三阶段：社区版本（v1.2）

### 阶段目标

提升功能深度，满足高级用户需求。

### 功能清单

#### F1：高级校验

- 导表前检查 xlsx 格式（字段类型匹配、必填项）
- 错误定位到具体单元格（行列坐标）
- 生成校验报告

#### F2：导表历史

- 保存导表历史（时间、项目、配置、结果）
- 可查看历史日志
- 快速回溯到历史配置

#### F3：日志增强

- 日志搜索和过滤（按级别、关键字）
- 导出日志为文本文件
- 统计信息（导出记录数、耗时）

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| ClosedXML 生成格式与 Luban 不兼容 | 中 | 高 | 早期用 Luban CLI 验证生成的 xlsx，固化测试用例 |
| COM 自动化在某些 Excel 版本不可用 | 中 | 低 | 退化为"打开文件"，不影响核心功能 |
| Luban 版本更新导致参数不兼容 | 低 | 中 | 内置固定版本 Luban，文档注明版本号 |
| 进程通信死锁 bug | 中 | 高 | 早期充分测试，并行读取 stdout/stderr，单元测试覆盖 |
| Excel 路径含空格/中文 | 中 | 中 | 路径参数加引号 (`"path"`），边界测试覆盖 |

## 八、个人开发优先级策略

### 分级

**必须做（v1.0，不可跳过）**：
- 项目管理（新建/打开/切换）
- 快捷操作（新建表格/枚举/Bean）
- 表格预览
- 简化导出配置
- 一键导表 + 实时日志

**应该做（v1.1，视进度安排）**：
- 多导出配置预设
- 增量导表
- 单元测试

**可以做（v1.2+，后期按需）**：
- 高级校验
- 导表历史
- 在线字段编辑（v2.0）

### 原则

- **每周有可运行的版本**：宁可功能不完善，不能无法编译，随时可以演示
- **先主线后分支**：核心导表链路优先，UI 美化和边界情况其次
- **尽早验证**：Week 2 结束前验证 ClosedXML 生成的 xlsx 能被 Luban 识别
