# Luban-GUI v0.2 开发计划

**版本**: v0.2  
**日期**: 2026年4月  
**状态**: 计划中

---

## 一、版本目标

基于 v0.1 预览版，补齐 Schema 可用性与编辑效率核心能力，形成“可创建类型、可审查类型、可快速定位源表、可稳定导表”的闭环。

本版本聚焦以下五项能力：

- 自定义类型：枚举、Bean
- 容器类型支持
- 数据类型总览（内置默认类型 + 自定义类型）
- 预览单元格右键定位到 Excel 表格对应单元格
- 删除表格功能

---

## 二、范围定义

### 2.1 In Scope（v0.2 必做）

- 枚举与 Bean 的完整创建流程（UI、服务层、元数据写入、错误提示）
- 字段类型支持容器类型输入与校验（含嵌套限制）
- 数据类型总览能力：可查看内置类型与自定义类型（enum、bean）
- 预览定位能力：右键单元格可定位并打开对应 Excel 文件中的目标单元格
- 表格删除能力：更新元数据注册并处理文件删除策略
- 核心回归链路：预览、导表、日志可用

### 2.2 Out of Scope（v0.2 不做）

- 多态类型
- 完整数据校验器体系
- 批量表结构重构能力
- 右侧面板直接编辑数据并写回 xlsx

---

## 三、里程碑与迭代安排（4 周）

### Week 1：自定义类型（枚举/Bean）

**目标**：完成“创建类型 -> 注册元数据 -> 字段可引用”链路。

**任务**：

1. 统一枚举和 Bean 的命名规则、重复检查与错误反馈。
2. 完善 SchemaService 的 CreateEnumAsync 与 CreateBeanAsync 一致性（异常语义、日志、失败处理）。
3. 接入类型来源到字段类型选择，支持创建后即时可见。
4. 增加 __enums__.xlsx 与 __beans__.xlsx 的读写验证。

**验收标准**：

- [x] 可通过 UI 创建枚举与 Bean。
- [ ] 重名与非法命名可被明确阻止。
- [x] 新建类型可被字段正确引用并导表通过。

### Week 2：容器类型支持

**目标**：字段可表达常用容器类型且可被 Luban 正确识别。

**任务**：

1. 定义 v0.2 容器语法白名单（例如 list、set、map）与最大嵌套层级。
2. 在字段编辑 UI 增加容器类型辅助输入（模板或语法提示）。
3. 服务层补充容器类型合法性校验（基础类型、自定义类型、组合类型）。
4. 导表前对容器类型做前置校验并输出字段级错误信息。

**验收标准**：

- [x] 至少 3 类常用容器类型能稳定导表。
- [ ] 非法容器表达式会被拦截并定位到字段。
- [ ] 不影响原有基础类型字段流程。

### Week 3：类型总览与预览定位

**目标**：提升类型可见性与表格修改效率，减少字段类型填写和回查成本。

**任务**：

1. 增加“数据类型列表”视图，统一展示内置默认类型与自定义类型（enum、bean）。
2. 提供类型分类与来源标识（内置/自定义），便于审查 enum/bean 定义是否完整。
3. 在表格预览右键菜单新增“定位到 Excel单元格 表格”操作。
4. 建立预览单元格到 Excel 文件路径 + 行列坐标的映射，支持打开并精准定位。

**验收标准**：

- [ ] 可在类型列表中查看全部可用类型，并区分内置与自定义来源。
- [ ] 新建 enum/bean 后可在类型列表中即时可见。
- [ ] 在预览中右键任一可定位单元格，能打开对应 Excel 并定位到目标单元格。

### Week 4：删除表格与整体回归

**目标**：完成删除闭环并验证 v0.2 主链路稳定性。

**任务**：

1. 表格列表增加“删除表格”入口与二次确认。
2. 删除策略支持：仅移除注册，或同时删除物理文件（默认策略可配置）。
3. 同步更新 __tables__.xlsx，处理删除当前选中表时的 UI 状态。
4. 执行联测：自定义类型、容器类型、类型总览、预览定位、删除表格与导表。

**验收标准**：

- [ ] 删除后列表、预览与状态栏一致。
- [ ] __tables__.xlsx 注册项正确更新。
- [ ] 典型项目下全量导表通过，无阻断级缺陷。

## 四、技术实现约束

- 严格保持五层架构边界：ViewModel 编排，规则在 Service，文件写入在 Infrastructure。
- 所有结构变更统一经 Service 接口，不在 ViewModel 直接改 xlsx。
- 关键操作必须记录结构化日志（开始、成功、失败、耗时）。
- 类型与字段校验前置，减少运行时 CLI 报错。

## 五、测试与质量门禁

### 5.1 功能测试清单

- [ ] 枚举/Bean：成功、重名、非法名称。
- [ ] 容器类型：合法组合、非法嵌套、未知类型引用。
- [ ] 数据类型总览：内置类型展示、自定义类型展示、创建后即时刷新。
- [ ] 预览定位：右键菜单可见性、文件打开成功、单元格定位准确性。
- [ ] 删除表格：仅删注册、删文件、删除后导表。
- [ ] 回归：项目切换、预览、导表、日志窗口。

### 5.2 质量门禁

- 构建通过：dotnet build .\LubanGui.slnx
- 核心手工回归通过：主链路无阻断问题
- 文档同步完成：CHANGELOG 与开发计划一致

## 六、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 容器语法设计过宽导致进度失控 | 中-高 | 先采用白名单 + 层级限制，后续版本扩展 |
| 类型列表与实际可用类型不一致 | 中 | 类型源统一经 SchemaService 聚合，增加刷新与缓存失效策略 |
| 预览坐标映射错误导致定位偏移 | 高 | 明确定义行列映射规则并增加关键样例回归 |
| 删除表格引发引用断裂 | 高 | 删除前做引用检查并给阻断提示 |
| 自定义类型命名冲突 | 中 | 全局命名规则统一校验 |
