## 主窗口

主窗口由四个区域构成：**工具栏**、**配置面板**、**日志面板**、**状态栏**。  
配置面板与日志面板之间设有可拖动的分隔线，用户可自由调整各区域高度。

### 工具栏

位于窗口顶部，包含三类元素：

#### 操作按钮（左侧）

| 按钮 | 图标 | 快捷键 | 说明 |
|------|------|--------|------|
| 全量导表 | ▶ | `F5` | 验证配置后启动 Luban 进程，执行完整导表 |
| 校验配置 | ✅ | `F6` | 仅检查配置项是否合法，不执行导表 |
| 取消 | ⏹ | `Esc` | 中止正在进行的导表（仅导表中可用） |

**状态指示器**（按钮右侧）：
- `● 就绪` — 灰色，等待操作
- `⟳ 导表中…` — 蓝色旋转动画，显示已耗时秒数
- `✓ 成功` — 绿色，显示 3 秒后恢复为就绪
- `✗ 失败` — 红色，常驻直至用户再次操作

#### 辅助按钮（右侧）

| 按钮 | 说明 |
|------|------|
| ⚙ 设置 | 打开应用设置（主题、语言等，v1.1） |
| ? 关于 | 打开关于窗口 |

---

### 配置面板

配置面板是主界面的核心区域，采用**可折叠分组**布局，将参数按重要性分为三组。

#### 配置名称

面板顶部为一个文本输入框，用于填写当前配置的本地标识名称（不传给 Luban CLI），方便用户在多配置场景下区分（v1.1 功能）。

#### 必填参数组

默认展开，始终可见。

```
▸ 必填参数
  Luban 路径:  [路径输入框__________________________] [📂 浏览]
  配置文件:    [路径输入框__________________________] [📂 浏览]
  目标:        [文本输入框__________________________]
```

- **Luban 路径**：Luban 可执行文件路径，支持手动输入或通过"浏览"按钮选择文件
- **配置文件**：Luban `--conf` 参数对应的配置文件路径，支持"浏览"按钮
- **目标**：Luban `--target` 参数，如 `client`、`server`、`all`

> 三个字段均为必填，任何一项为空时"全量导表"和"校验配置"按钮置灰。

#### 常用可选参数组

默认展开，包含日常使用频率最高的可选参数。

```
▾ 常用可选
  代码目标:   [cs-bin ×] [typescript-json ×]  [+ 添加]
  数据目标:   [bin ×]                         [+ 添加]
  扩展参数:   [outputCodeDir=D:\Gen ×]        [+ 添加]
```

- **代码目标** (`--codeTarget`)：Tag 标签式多值输入，点击 `×` 删除，点击`+ 添加`弹出输入框
- **数据目标** (`--dataTarget`)：同上
- **扩展参数** (`--xargs`)：格式 `key=value`，输入时自动验证格式

#### 高级参数组

默认折叠，面向高级用户。

```
▸ 高级参数
  （点击展开）
```

展开后显示：

| 字段标签 | CLI 参数 | 控件类型 |
|---------|---------|---------|
| Schema 收集器 | `--schemaCollector` | 文本框 |
| Pipeline | `--pipeline` | 文本框 |
| 只输出指定表 | `--outputTable` | 多值标签输入 |
| 包含标签 | `--includeTag` | 多值标签输入 |
| 排除标签 | `--excludeTag` | 多值标签输入 |
| 字段变体 | `--variant` | 多值标签输入（key=value）|
| 时区 | `--timeZone` | 文本框 |
| 自定义模板目录 | `--customTemplateDir` | 多值路径输入 + 浏览 |
| 验证失败作为错误 | `--validationFailAsError` | 复选框 |
| 监听目录 | `--watchDir` | 多值路径输入 + 浏览 |
| 强制加载表数据 | `--forceLoadTableDatas` | 复选框 |
| Verbose 输出 | `--verbose` | 复选框 |
| NLog 配置文件 | `--logConfig` | 文本框 + 浏览 |

### 日志面板

位于配置面板下方，实时显示 Luban 进程的输出，是用户判断导表状态的核心区域。

```
┌─────────────────────────────────────────────────── [🗑 清空]─┐
│  [INFO]    15:00:01  就绪，等待导表...                        │
│  [INFO]    15:00:05  验证配置...                              │
│  [INFO]    15:00:05  启动 Luban 进程...                       │
│  [OUT]               Processing Excel files...               │
│  [OUT]               Sheet1: 100 rows                        │
│  [WARN]              Missing optional field at Sheet2!B3     │
│  [SUCCESS] 15:00:07  导表完成 (耗时 2.34s)                    │
└─────────────────────────────────────────────────────────────┘
```

#### 日志级别与颜色

| 前缀 | 含义 | 颜色 |
|------|------|------|
| `[INFO]` | 一般流程信息（GUI 自身产生） | 灰色 |
| `[OUT]` | Luban stdout 原始输出 | 白色/默认色 |
| `[WARN]` | 警告信息（来自 Luban stderr 或 GUI 判断） | 黄色 |
| `[ERROR]` | 错误信息 | 红色 |
| `[SUCCESS]` | 导表成功（GUI 在进程退出码为 0 时追加） | 绿色 |

#### 面板功能

- **自动滚动**：默认跟随最新日志；用户手动向上滚动时暂停自动滚动，日志区底部出现"↓ 跳到最新"按钮
- **清空**：点击右上角🗑按钮清空所有日志（每次新的导表开始前也会自动清空）
- **全选 + 复制**：支持 `Ctrl+A` 全选、`Ctrl+C` 复制，方便用户粘贴错误信息到 Issue 或聊天工具
- **虚拟化渲染**：支持 10,000+ 条日志不卡顿

### 状态栏

位于窗口最底部，显示持久化状态信息。

## 交互行为规范

### 按钮状态

| 状态 | 全量导表 | 校验配置 | 取消 |
|------|---------|---------|------|
| 初始（必填项为空） | 置灰 | 置灰 | 置灰 |
| 就绪（必填项已填） | 可用 | 可用 | 置灰 |
| 导表进行中 | 置灰 | 置灰 | 可用 |
| 导表完成 | 可用 | 可用 | 置灰 |

### 路径输入框

- 支持直接拖拽文件/文件夹到输入框
- 路径不存在时，输入框边框高亮为红色，Tooltip 显示具体错误
- 路径有效时，输入框边框为正常色

### 多值标签输入（Tag Input）

用于 `--codeTarget`、`--dataTarget`、`--xargs` 等多值参数：

```
[cs-bin ×]  [typescript-json ×]   [+ 添加]
```

- 点击`+ 添加`弹出内联输入框，回车确认，`Escape` 取消
- 点击标签上的`×`删除该项
- `--xargs` 和 `--variant` 类型的输入框在失焦时验证 `key=value` 格式，不符合时显示红色边框

### 校验反馈

点击"校验配置"后：
- **通过**：在工具栏状态区显示绿色 `✓ 配置有效`，3 秒后恢复
- **失败**：在工具栏状态区显示红色 `✗ 配置错误`，并在日志面板追加具体错误列表：
  ```
  [ERROR] 校验失败，发现以下问题：
  [ERROR]   • Luban 路径不存在: C:\NoSuch\luban.exe
  [ERROR]   • xargs 格式错误: "badformat"（应为 key=value）
  ```

## 视觉规范

### 主题

- v1.0 仅提供**浅色主题**（Avalonia 默认）
- v1.1 计划引入暗色主题切换

### 颜色语义

| 用途 | 颜色 |
|------|------|
| 成功、正常 | `#4CAF50`（绿色）|
| 错误 | `#F44336`（红色）|
| 警告 | `#FF9800`（橙黄色）|
| 进行中 | `#2196F3`（蓝色）|
| 禁用/次要 | `#9E9E9E`（灰色）|

### 布局尺寸

- **最小窗口尺寸**：宽 700px × 高 600px
- **默认窗口尺寸**：宽 900px × 高 750px
- **配置面板 / 日志面板** 初始高度比例：60% / 40%，可拖拽分隔线调整
- **工具栏**：高度 48px，按钮内边距 8px × 16px

## 后续版本规划

### v1.1：多配置支持

在主窗口左侧新增**配置列表面板**，支持管理多套导表配置（对应不同项目或目标）。

```
┌──────────────────────────────────────────────────────────────────────┐
│  TOOLBAR                                                             │
├──────────────────┬───────────────────────────────────────────────────┤
│  📁 配置列表     │  ⚙️  DefaultGame                                 │
│  ┌────────────┐  │  ┌──── 配置面板 ───────────────────────────────┐  │
│  │▶DefaultGame│  │  │  （同 v1.0 三组折叠布局）                  │  │
│  │  Tools     │  │  └─────────────────────────────────────────────┘  │
│  │            │  │                                                   │
│  │[+新建][-]  │  │  ┌──── 日志面板 ──────────────────────────────┐  │
│  └────────────┘  │  │                                             │  │
│                  │  └─────────────────────────────────────────────┘  │
└──────────────────┴───────────────────────────────────────────────────┘
```

- 配置列表支持新建、复制、删除、重命名
- 支持配置文件导入/导出（JSON 格式）
- 切换配置时，配置面板内容同步刷新

### v1.2：日志增强

- 日志面板顶部增加过滤工具栏（按级别筛选、关键字搜索）
- 支持将日志导出为文本文件
- 显示导表历史记录（可点击查看历史日志）
