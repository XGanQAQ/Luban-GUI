**完成时间**: 4 周（个人全职开发估计为主）

**目标**: 实现完整的基础导表功能，足以满足大多数日常使用场景。

**完成标准**:
- ✅ 核心功能全部完成（配置、导表、日志、校验）
- ✅ 没有致命 bug
- ✅ 代码可读、有注释
- ✅ 有初步的用户文档

### 阶段任务分解

#### Week 1: 项目框架搭建

**目标**: 建立可运行的项目框架，核心架构确定

**任务**:

1. ✅ **项目初始化**
   - 使用 Avalonia 模板创建项目
   - 配置 .csproj，添加必需的 NuGet 包
   - 验证项目编译和运行

2. ✅ **依赖注入配置**
   - 在 App.xaml.cs 中配置 DI 容器
   - 注册所有 Service、ViewModel、View
   - 验证依赖注入能正常工作

3. ✅ **日志系统初始化**
   - 配置 Serilog
   - 设置日志输出到控制台和文件
   - 创建 Logger 工厂方法

4. ✅ **UI 布局基础**（详见 [04-前端界面文档.md](./04-前端界面文档.md)）
   - 主窗口分四区域：工具栏、配置面板、日志面板、状态栏
   - 配置面板使用可折叠分组（必填 / 常用可选 / 高级参数）
   - 工具栏包含：全量导表、校验配置、取消、状态指示器、关于按钮
   - 日志面板支持虚拟化滚动、自动滚动、清空、全选复制
   - 状态栏显示上次导表结果和版本号

**验收标准**:
- [ ] 项目编译通过，无编译错误
- [ ] 应用能启动，显示主窗口
- [ ] DI 容器能正常注入依赖
- [ ] Serilog 能正常输出日志
- [ ] UI 布局基本完成（美化可后续迭代）

**本周风险**: 无特殊风险，都是熟悉的库

#### Week 2: 核心服务层开发

**目标**: 实现业务逻辑层和工具层，使导表流程可以跑通

**任务**:

1. ✅ **数据模型定义**
   - ExportConfig（配置模型，字段对应 Luban CLI 参数）
   - ExportResult（结果模型）
   - LogEntry（日志模型）
   - 实现模型验证方法（必填字段：LubanPath、ConfFile、Target）

2. ✅ **ConfigManager 实现**
   - 配置文件路径确定（AppData\Local）
   - 实现 LoadConfigAsync（从文件读）
   - 实现 SaveConfigAsync（保存到文件）
   - 实现 ValidateConfig（验证方法：路径存在、必填项非空）
   - **测试**: 配置能正确读写，应用重启后能加载

3. ✅ **LubanCommandBuilder 实现**
   - 根据 ExportConfig 构建完整的命令行参数（--conf、--target 等）
   - 支持所有 Luban CLI 参数，包括可重复参数（codeTarget、dataTarget、xargs 等）
   - 参数验证（ConfFile/LubanPath 路径存在性检查）
   - **测试**: 输入不同配置，验证生成的参数字符串正确

4. ✅ **ProcessManager 实现（重要！）**
   - 创建进程，配置 ProcessStartInfo
   - 实现异步输出捕获（核心技术）
     - 异步读取 stdout
     - 异步读取 stderr
     - 使用 Task.WhenAll 避免死锁
   - 实现 RunAsync 方法
   - **测试**: 能成功启动记事本，捕获其输出

5. ✅ **LubanExecutor 实现**
   - 整合 CommandBuilder 和 ProcessManager
   - 实现 ExecuteExportAsync
   - 添加日志输出（参数、启动、完成）
   - **测试**: 能调用 Luban（如果安装了），显示日志

6. ⚠️ **ExportService 实现**
   - 编排整个导表流程
   - 验证配置 → 创建输出目录 → 调用 Executor → 保存配置
   - 异常处理
   - **测试**: 端到端测试整个流程

**验收标准**:
- [ ] 配置能正确读写
- [ ] 命令行参数构建正确
- [ ] ProcessManager 能启动进程并捕获输出
- [ ] 整个导表流程能跑通（使用虚拟的 Luban.exe 或真实的 Luban）
- [ ] 无内存泄漏（进程正确释放）

**本周难点**: ProcessManager 的异步输出捕获，可能需要花时间调试

**参考资源**:
- System.Diagnostics.Process 文档
- Task 并行编程指南
- IProgress<T> 模式

#### Week 3: UI 实现和绑定

**目标**: 完成 UI 界面和 ViewModel，使用户能与应用交互

**任务**:

1. ✅ **ViewModel 实现**
   - MainWindowViewModel 继承 ViewModelBase
   - 属性绑定:
     - Config（配置对象，包含所有 CLI 参数字段）
     - Logs（日志集合，ObservableCollection<string>）
     - IsExporting（是否正在导表）
     - StatusMessage（状态提示）
   - 命令实现:
     - ExportCommand（全量导表）
     - ValidateCommand（校验配置）
     - BrowseLubanCommand（浏览 Luban 可执行文件）
     - BrowseConfFileCommand（浏览 --conf 配置文件）
     - BrowseCustomTemplateDirCommand（浏览自定义模板目录）
   - **关键**: 命令执行时需要禁用按钮（IsExporting），导表完成后启用

2. ✅ **View 完善**
   - 必填配置面板（LubanPath、ConfFile、Target）
   - 常用可选面板（CodeTargets、DataTargets、Xargs 列表编辑）
   - 高级选项折叠面板（SchemaCollector、Pipeline、IncludeTags、ExcludeTags 等）
   - 操作面板
     - 【全量导表】按钮，IsEnabled="{Binding !IsExporting}"
     - 【校验】按钮
     - 状态标签，Text="{Binding StatusMessage}"
   - 日志窗口
     - ListBox 显示 Logs（虚拟化）
     - 颜色着色（根据 [INFO]/[ERROR] 等前缀）

3. ✅ **命令处理逻辑**
   - 全量导表: 
     ```
     1. 设置 IsExporting=true （按钮禁用）
     2. 清空 Logs
     3. 创建 IProgress<string>，捕获日志到 Logs
     4. 调用 ExportService.ExportAllAsync()
     5. 显示结果（成功/失败）
     6. 设置 IsExporting=false （按钮启用）
     ```
   - 校验: 验证配置，显示错误列表
   - 浏览: 打开文件夹对话框，更新配置

4. ✅ **日志窗口优化**
   - 实现着色（[INFO] 灰色，[SUCCESS] 绿色，[ERROR] 红色）
   - 自动滚动到最新（当新日志添加时）
   - 支持全选和复制

**验收标准**:
- [ ] 可以在 UI 中输入配置
- [ ] 文件夹浏览对话框能打开并选择目录
- [ ] 【校验】按钮能验证配置，显示错误
- [ ] 【全量导表】按钮能触发导表流程
- [ ] 日志实时显示，不卡顿
- [ ] 导表时【全量导表】按钮被禁用
- [ ] 导表完成后有清晰的成功/失败提示

**本周重点**: 确保数据绑定正确，特别是日志的实时更新

## 第二阶段: 功能优化

### 阶段目标

**完成时间**: 2 周（v1.0 发布后）

**目标**: 提升用户体验，支持多项目管理，初步支持增量导表

**版本**: v1.1

### 实现功能

#### F1: 多配置管理

**需求**: 用户可以为不同项目保存不同的配置，快速切换

**实现**:
- 配置列表视图（左侧或顶部下拉）
- 当前配置: "GameProject" / "ToolProject" 等
- 按钮: 保存为新配置、删除配置、导出配置、导入配置
- 存储: 配置目录下保存多个 config_*.json 文件

**UI 更改**:
```
┌─ 配置切换 ──────────┐
│ 项目: [GameProject ▼]│
│ [新建] [删除] [导出]  │
└──────────────────────┘
```

#### F2: 增量导表基础

**需求**: 检测自上次导表后修改的 Excel 文件，只导出修改的表

**实现**:
- 记录上次导表的时间
- 比对 Excel 文件的修改时间
- 导表前显示"将导出 X 个修改的表"
- 可选: 支持"强制全量导表"

**数据**:
```csharp
public class ExportConfig
{
    public DateTime LastExportTime { get; set; }
    public bool UseIncrementalExport { get; set; }
}
```

#### F3: UI 优化

- 暗黑模式支持
- 记住窗口大小和位置
- 快捷键支持（Ctrl+S 快速导表）
- 任务栏进度显示（导表进度）
