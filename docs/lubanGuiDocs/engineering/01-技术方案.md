## 一、技术选型

### 技术栈

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| **UI 框架** | Avalonia | 11.x | 开源跨平台 XAML 框架，C# 生态，MVVM 支持完善 |
| **编程语言** | C# | 10+ | 类型安全，LINQ，现代语言特性 |
| **MVVM 工具包** | CommunityToolkit.Mvvm | 8.x | 源生成器减少样板代码，`ObservableObject`、`[RelayCommand]` |
| **配置读写** | System.Text.Json | 内置 | 内置库，无需第三方依赖，性能好，AOT 友好 |
| **Excel 操作** | ClosedXML | 0.100+ | 无需安装 Excel，纯代码操作 xlsx，API 清晰 |
| **进程通信** | System.Diagnostics.Process | 内置 | 标准库，稳定可靠，支持重定向 stdout/stderr |
| **日志** | Serilog | 3.x | 结构化日志，支持多 Sink（文件 + 控制台），异步写入 |
| **依赖注入** | Microsoft.Extensions.DependencyInjection | 8.0 | 标准库，Avalonia 原生支持 |

### 为什么选 Avalonia

✅ **开源免费** — MIT 许可，无商业限制  
✅ **跨平台** — 同一套代码支持 Windows / Linux / macOS  
✅ **XAML 支持** — 熟悉 WPF 的开发者易上手  
✅ **完整 UI 库** — Dialog、DataGrid、ListBox 等常用控件齐全  
✅ **社区活跃** — 问题容易找到答案  
✅ **C# 生态** — 与 .NET 框架完美集成

### 为什么不选 WPF

❌ Windows only，无法跨平台  
❌ 官方已不推荐新项目使用  
❌ 渲染性能弱于 Avalonia

### 为什么不选 WinUI 3

❌ 学习成本高，生态不成熟  
❌ 对个人开发者不友好

### 为什么选 System.Text.Json

✅ **内置库** — 零外部依赖  
✅ **性能好** — 比 Newtonsoft.Json 快  
✅ **AOT 友好** — 支持 Native AOT 优化

### 为什么选 Serilog

✅ **结构化日志** — 便于后续分析  
✅ **灵活 Sink** — 支持文件、控制台、数据库等  
✅ **异步写入** — 不阻塞主线程  
✅ **社区成熟** — NuGet 下载量最高

## 二、整体架构

### 设计定位

LubanGui **不是** Luban CLI 的简单参数拼装器，而是一个拥有自身业务逻辑的**表格项目管理工具**。它：

1. 维护项目工作区（`luban.conf`、`Datas/`、元数据表）
2. 提供快捷操作（按 Luban 规范自动生成 xlsx）
3. 内置数据预览（无需打开 Excel）
4. 封装 Luban 全部 CLI 参数，对用户透明

### 五层架构

```
┌──────────────────────────────────────────────────────┐
│               表现层 (Avalonia UI)                   │
│  主窗口 · 项目切换 · 快捷操作 · 导出配置 · 日志窗口   │
│  Views (AXAML) + ViewModels (MVVM)                   │
└────────────────────────┬─────────────────────────────┘
                         │ 接口调用（向下单向依赖）
┌────────────────────────▼─────────────────────────────┐
│              业务逻辑层 (Service)                     │
│  ProjectManager · SchemaService                      │
│  ExportService · TablePreviewService                 │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│                工具层 (Executor / IO)                 │
│  LubanExecutor · LubanCommandBuilder                 │
│  ExcelWriter · FileOpenService                       │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│              基础设施层 (Infrastructure)              │
│  AppConfigManager · ProjectConfigManager · Logger    │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│           Luban 源适配层 (LubanAdapter)               │
│  LubanSchemaReader · LubanConfAdapter                │
│  LubanTypeMapper                                     │
└──────────────────────────────────────────────────────┘
```

> **约束**：层间只向下依赖。Luban 源适配层是 GUI 与 `lubanSrc\` 之间的**唯一接触点**，上方各层不得直接引用 Luban 源码中的类型。

### 项目工作区结构

```
<WorkspaceRoot>/
└── <ProjectName>/
    ├── projectConfig.json        # GUI 专属导出配置
    ├── luban.conf                # Luban 全局配置（由 GUI 自动维护）
    ├── Defines/                  # 自定义类型定义
    └── Datas/
        ├── __tables__.xlsx       # 表格元数据（由 GUI 维护）
        ├── __beans__.xlsx        # Bean 元数据（由 GUI 维护）
        ├── __enums__.xlsx        # 枚举元数据（由 GUI 维护）
        └── *.xlsx                # 用户数据表
```

## 三、分层说明

> 详细的层次职责说明见 [02-层次划分.md](./02-层次划分.md)

### 表现层（Avalonia UI）

**职责**：UI 显示与用户交互，路由命令到 ViewModel。

**主要 ViewModel**：

| ViewModel | 职责 |
|-----------|------|
| `MainWindowViewModel` | 统筹项目切换、表格列表、表格预览、导表状态 |
| `TableListViewModel` | 管理左侧表格列表，响应快捷操作命令 |
| `TablePreviewViewModel` | 管理右侧表格数据预览，响应双击定位命令 |
| `ExportConfigViewModel` | 管理导出配置（数据类型/路径、代码类型/路径） |
| `LogWindowViewModel` | 管理日志浮动窗口 |

> 详细 UI 布局见 [02-前端界面设计.md](./02-前端界面设计.md)。

---

### 业务逻辑层（Service）

**职责**：实现核心业务流程，协调工具层完成复杂操作。

| 服务 | 职责 |
|------|------|
| `ProjectManager` | 管理多个项目的创建、切换、加载，初始化工作区 |
| `SchemaService` | 维护 Luban 元数据表，提供创建新表/枚举/Bean 操作 |
| `ExportService` | 根据 `ProjectConfig` 编排完整的导表流程 |
| `TablePreviewService` | 读取 Excel 数据文件，为右侧面板提供预览数据 |

---

### 工具层（Executor / IO）

**职责**：封装底层 IO 和进程操作，供 Service 调用。

| 组件 | 职责 |
|------|------|
| `LubanExecutor` | 启动内置 Luban CLI 子进程，异步捕获 stdout/stderr |
| `LubanCommandBuilder` | 将 `ProjectConfig` 映射为完整的 Luban CLI 参数（内部不暴露给用户） |
| `ExcelWriter` | 按 Luban 规范生成和写入 xlsx（创建表格/枚举/Bean 时使用） |
| `FileOpenService` | 打开 Excel 文件并定位到指定字段（双击列时触发） |

---

### 基础设施层（Infrastructure）

**职责**：持久化配置，提供日志等横切关注点。

| 组件 | 职责 | 存储位置 |
|------|------|---------|
| `AppConfigManager` | 全局应用配置（已注册项目列表、上次打开项目） | `%LOCALAPPDATA%\LubanGui\appConfig.json` |
| `ProjectConfigManager` | 单个项目的导出配置读写 | `<ProjectPath>/projectConfig.json` |
| `Logger` (Serilog) | 统一日志管理，写入文件和控制台 | `logs/luban-gui-<日期>.txt` |

---

### Luban 源适配层（LubanAdapter）

**职责**：隔离 GUI 与 `lubanSrc\` 之间的耦合，封装对 Luban 内部数据结构的读取。

| 组件 | 职责 |
|------|------|
| `LubanSchemaReader` | 读取 `luban.conf` 中的 targets 列表，供导出配置窗口使用 |
| `LubanConfAdapter` | 生成并维护 `luban.conf` 文件内容 |
| `LubanTypeMapper` | 维护 Luban 内置类型（int/string/bool/float 等）供 GUI 展示 |

## 四、核心模块接口

### IProjectManager

```csharp
public interface IProjectManager
{
    IReadOnlyList<ProjectInfo> Projects { get; }
    ProjectInfo? CurrentProject { get; }

    // 在指定路径下初始化工作区
    Task<ProjectInfo> CreateProjectAsync(string name, string workspacePath);
    // 打开已有项目文件夹
    Task<ProjectInfo> OpenProjectAsync(string workspacePath);
    // 切换当前项目
    Task SwitchProjectAsync(string projectName);
    // 从列表移除（不删除文件）
    void RemoveProject(string projectName);
}
```

**工作区初始化流程（CreateProjectAsync）**：

```
1. 在 workspacePath 下创建 <name>/ 目录
2. 生成初始 luban.conf（包含默认 groups、schemaFiles、targets）
3. 生成初始 Datas/__tables__.xlsx、__beans__.xlsx、__enums__.xlsx
4. 生成初始 projectConfig.json（空的输出配置）
5. 将项目信息写入全局 appConfig.json
```

---

### ISchemaService

```csharp
public interface ISchemaService
{
    // 读取当前项目所有已注册表格
    Task<IReadOnlyList<TableMeta>> GetTablesAsync();

    // 生成符合 Luban 规范的新 xlsx，并写入 __tables__.xlsx
    Task<TableMeta> CreateTableAsync(NewTableOptions options);
    // 注册已有 xlsx 到 __tables__.xlsx
    Task<TableMeta> ImportTableAsync(string xlsxPath);
    // 写入 __enums__.xlsx
    Task CreateEnumAsync(NewEnumOptions options);
    // 写入 __beans__.xlsx
    Task CreateBeanAsync(NewBeanOptions options);
    // 从 __tables__.xlsx 删除一条记录
    Task RemoveTableAsync(string tableName);
}
```

**新建表格时 xlsx 格式（符合 Luban 约定）**：

```
行1（A=##var）:   id      name     value    desc
行2（A=##type）:  int     string   int      string
行3（A=##group）:          (group)
行4（A=##）:      id      名称     数值     说明
行5（数据行）:    1       示例     100      说明
```

### IExportService

```csharp
public interface IExportService
{
    // 执行完整导表流程
    Task<ExportResult> ExportAsync(
        ProjectConfig config,
        IProgress<string> logProgress,
        CancellationToken cancellationToken = default);

    // 仅验证配置（不执行导表）
    ValidationResult ValidateConfig(ProjectConfig config);
}
```

**导表流程**：

```
1. ValidateConfig（路径非空、类型有效）
2. 确保输出目录存在（Directory.CreateDirectory）
3. ProjectConfig → LubanRunArgs（内部映射，用户不可见）
4. LubanExecutor 执行 Luban CLI
5. 保存 ProjectConfig 到 projectConfig.json
6. 返回 ExportResult
```

**异常类型**：

| 异常 | 含义 |
|------|------|
| `ConfigInvalidException` | 配置校验失败 |
| `ProcessException` | Luban 进程运行时错误 |
| `OperationCanceledException` | 用户取消 |

### LubanCommandBuilder（内部，不暴露给用户）

**职责**：将 `ProjectConfig` 转换为完整的 Luban CLI 参数。

| ProjectConfig 字段 | 对应 Luban CLI 参数 |
|-------------------|-------------------|

## 五、关键技术方案

### 方案 1：线程安全的 UI 更新

**问题**：`LubanExecutor` 在后台线程读取进程输出，如何安全地更新 UI？  
**方案**：使用 `IProgress<T>`，内部自动切换到 UI 线程。

```csharp
// ViewModel 中创建 Progress（自动在 UI 线程执行回调）
var progress = new Progress<string>(line => LogItems.Add(line));

// 后台线程中调用（线程安全）
logProgress?.Report(logLine);
```

---

### 方案 2：实时日志显示

**问题**：等进程完成再显示日志，用户体验差。  
**方案**：逐行异步读取，实时推送。

```
Luban 进程
  → stdout/stderr → StreamReader.ReadLineAsync()（逐行异步）
  → IProgress.Report()（线程安全推送到 UI）
  → LogItems.Add()（UI 线程更新列表）
```

**性能优化**：虚拟化列表渲染；限制最多保留 10000 行日志；异步写磁盘。

---

### 方案 3：进程输出捕获

**关键 ProcessStartInfo 设置**：

```csharp
var psi = new ProcessStartInfo
{
    RedirectStandardOutput = true,
    RedirectStandardError  = true,
    UseShellExecute        = false,      // 不通过 cmd.exe
    CreateNoWindow         = true,       // 不创建控制台窗口
    StandardOutputEncoding = Encoding.UTF8,
    StandardErrorEncoding  = Encoding.UTF8,
};
```

---

### 方案 4：避免进程通信死锁

**问题**：若只读一个流，另一个流缓冲区满时会死锁。  
**方案**：并行异步读取 stdout 和 stderr 两个流。

```csharp
var stdoutTask = Task.Run(() => ReadStreamAsync(process.StandardOutput, logProgress, "OUT"));
var stderrTask = Task.Run(() => ReadStreamAsync(process.StandardError,  logProgress, "ERR"));
await Task.WhenAll(stdoutTask, stderrTask);
await process.WaitForExitAsync(cancellationToken);
```

## 六、架构优势

| 优势 | 说明 |
|-----|------|
| **用户友好** | 用户只关心"导出什么类型、输出到哪里"，无需了解 CLI 参数 |
| **项目完整性** | GUI 负责维护 `luban.conf` 和元数据表，不会出现手动配置遗漏 |
| **关注分离** | 每层只关心自己的职责，不关心其他层的实现 |
| **易于测试** | 可以独立测试每一层，Mock 其他层 |
| **天然扩展点** | 新增输出格式时，只需扩展 `ProjectConfig` 枚举值和 `CommandBuilder` 映射 |
| **轻量级** | 最小化外部依赖，ClosedXML 是唯一新增的核心依赖 |
| **跨平台** | 可编译到 Windows / Linux / macOS（文件定位在非 Windows 环境退化为"打开文件"） |

## 八、开发环境

### 必装软件

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| .NET SDK | 8.0+ | 编译和运行 |
| Visual Studio 2022 或 Rider | 任意最新版 | IDE |
| Avalonia 扩展 | 最新版 | AXAML 设计器支持 |

### 编译和运行

```bash
# 构建 GUI（含内嵌的 Luban CLI）
dotnet build .\LubanGui.slnx

# 仅构建 Luban 工具链
dotnet build .\lubanSrc\Luban.sln

# 运行 GUI
dotnet run --project .\LubanGui\LubanGui.csproj

# 发布（自包含可执行文件）
dotnet publish .\LubanGui\LubanGui.csproj -c Release -o .\publish
```

### 代码风格

遵循 `lubanSrc\.editorconfig`：Allman 大括号风格，强制使用大括号，最大行长 180。
