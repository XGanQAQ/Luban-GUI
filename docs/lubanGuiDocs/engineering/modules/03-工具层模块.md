# 工具层模块设计

**所属层次**: 工具层 (Executor / IO)  
**版本**: 2.0

---

## 职责

封装底层 IO 操作和进程管理，不包含任何业务逻辑。工具层的每个组件只做一件事，供业务逻辑层按需组合调用。

---

## LubanExecutor

**职责**: 启动 Luban CLI 子进程，异步捕获 stdout/stderr 输出流，并通过 `IProgress<string>` 实时推送到调用方

**进程配置**:
```csharp
var psi = new ProcessStartInfo
{
    RedirectStandardOutput = true,      // ✓ 重定向 stdout
    RedirectStandardError = true,       // ✓ 重定向 stderr
    UseShellExecute = false,            // ✓ 不通过 cmd.exe
    CreateNoWindow = true,              // ✓ 不创建窗口
    StandardOutputEncoding = Encoding.UTF8  // ✓ UTF-8 编码
};
```

**死锁防护**:  
同时启动两个异步任务分别读取 stdout 和 stderr，避免缓冲区满时单流读取造成死锁：
```csharp
Task<string> outTask = ReadStreamAsync(process.StandardOutput);
Task<string> errTask = ReadStreamAsync(process.StandardError);
await Task.WhenAll(outTask, errTask);
```

**输出推送**:  
逐行读取输出并通过 `IProgress<string>` 实时推送，UI 线程自动切换：
```
Luban 进程
  ├─ stdout → ReadLineAsync() → IProgress.Report() → UI 更新
  └─ stderr → ReadLineAsync() → IProgress.Report() → UI 更新
```

## LubanCommandBuilder（内部，不暴露给用户）

**职责**: 将 `ProjectConfig` 转换为完整的 Luban CLI 参数字符串

**映射规则**:

| ProjectConfig 字段 | 转换为 Luban CLI 参数 |
|-------------------|----------------------|
| `project.WorkspacePath + "/luban.conf"` | `--conf <path>` |
| `config.Target` | `--target <target>` |
| `config.CodeOutput.Type` | `--codeTarget <type>` |
| `config.CodeOutput.OutputPath` | `--xargs outputCodeDir=<path>` |
| `config.DataOutput.Type` | `--dataTarget <type>` |
| `config.DataOutput.OutputPath` | `--xargs outputDataDir=<path>` |

> 此类仅供 `ExportService` 内部使用，不作为独立接口暴露。

## ExcelWriter

**职责**: 按 Luban 规范生成和写入 xlsx 文件，在创建新表/枚举/Bean 时使用

**依赖**: `ClosedXML` 库

**操作类型**:

| 操作 | 说明 |
|------|------|
| 新建数据表 xlsx | 写入正确的 `##` 元数据行（字段名、类型、分组） |
| 追加 `__tables__.xlsx` 条目 | 在元数据表末尾写入新行 |
| 追加 `__enums__.xlsx` 条目 | 在枚举元数据表末尾写入新行 |
| 追加 `__beans__.xlsx` 条目 | 在 Bean 元数据表末尾写入新行 |

**新建表格时写入的格式**（符合 Luban 约定）:
```
行1: ##  id      name     value    desc
行2: ##  int     string   int      string
行3: ##          (group)
行4:     1       示例     100      示例说明
```

## FileOpenService

**职责**: 打开 Excel 文件并可选地定位到指定字段（双击列标头时触发）

**接口**:
```csharp
public interface IFileOpenService
{
    // 用系统默认程序打开文件
    void OpenFile(string filePath);

    // 尝试通过 COM 自动化定位到指定单元格
    // 仅 Windows + Excel 可用时生效，否则退化为 OpenFile
    void OpenFileAtCell(string filePath, string cellAddress);
}
```

**平台兼容性**:

| 平台 | `OpenFileAtCell` 行为 |
|------|-----------------------|
| Windows + Excel 已安装 | 通过 COM 自动化打开并跳转到指定单元格 |
| Windows + 无 Excel | 退化为 `OpenFile`（系统默认程序打开） |
| Linux / macOS | 退化为 `OpenFile` |

---

## 层间约定

- 工具层组件不持有任何项目状态，所有操作通过参数传入。
- 工具层不调用业务逻辑层（单向依赖）。
- 每个工具类只做一件事，保持单一职责。
