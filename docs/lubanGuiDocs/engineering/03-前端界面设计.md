## 一、整体布局概览

主窗口采用**菜单栏 + 项目工具栏 + 左右分栏**布局：左侧为表格列表（含快捷操作），右侧为选中表格的内容预览。日志输出独立为浮动二级窗口，不占用主界面空间。

## 二、主窗口

### 菜单栏

位于窗口顶部，悬停/点击展开下拉菜单。右侧显示当前操作状态指示器。

#### 菜单结构

**文件 (F)**

| 菜单项 | 快捷键 | 说明 |
|--------|--------|------|
| 新建项目... | `Ctrl+N` | 创建新项目（选择工作区路径） |
| 打开项目... | `Ctrl+O` | 打开已有项目文件夹 |
| ─── | | |
| 退出 | `Alt+F4` | 退出应用程序 |

**操作 (A)**

| 菜单项 | 快捷键 | 说明 |
|--------|--------|------|
| ▶ 全量导表 | `F5` | 验证配置后启动 Luban，执行完整导表 |
| ✅ 校验配置 | `F6` | 仅检查导出配置是否合法，不执行导表 |
| ─── | | |
| ⏹ 取消 | `Esc` | 中止正在进行的导表（仅导表中可用） |
| ─── | | |
| ⚙ 导出配置... | `Ctrl+,` | 打开导出配置窗口 |

**视图 (V)**

| 菜单项 | 快捷键 | 说明 |
|--------|--------|------|
| 📋 日志窗口 | `Ctrl+L` | 打开/显示日志浮动窗口 |
| ─── | | |
| 刷新表格列表 | `F5` | 重新扫描当前项目工作区的表格文件 |

**帮助 (H)**

| 菜单项 | 说明 |
|--------|------|
| 关于 Luban-GUI | 打开关于窗口 |
| 查看文档 | 打开在线文档 |
| 在 GitHub 上查看 | 打开项目仓库页面 |

#### 状态指示器（菜单栏右侧）

| 样式 | 含义 |
|------|------|
| `● 就绪` — 灰色 | 等待操作 |
| `⟳ 导表中… 3s` — 蓝色旋转 | 正在执行，实时显示已耗时秒数 |
| `✓ 成功` — 绿色 | 导表完成，3 秒后恢复为就绪 |
| `✗ 失败` — 红色 | 导表失败，常驻直至用户再次操作 |

### 项目切换栏

```
┌──────────────────────────────────────────────────────────────────────┐
│  项目: [MyGame ▾]   [+ 新建项目]  [📂 打开项目文件夹]               │
└──────────────────────────────────────────────────────────────────────┘
```

| 控件 | 行为 |
|------|------|
| **项目下拉框** | 列出所有已注册项目，点击切换。切换后表格列表和内容区同步刷新 |
| **[+ 新建项目]** | 弹出对话框，输入项目名称并选择工作区路径，GUI 自动初始化工作区 |
| **[📂 打开项目文件夹]** | 在系统文件管理器中打开当前项目目录，方便拖入 Excel 文件 |

---

### 快捷操作工具栏

```
┌──────────────────────────────────────────────────────────────────────┐
│  [+ 新建表格]  [+ 新建枚举]  [+ 新建Bean]  [📥 导入文件]            │
└──────────────────────────────────────────────────────────────────────┘
```

| 按钮 | 行为 |
|------|------|
| **+ 新建表格** | 弹出对话框，输入表名和字段，GUI 自动在 `Datas/` 下生成符合 Luban 规范的 xlsx，并写入 `__tables__.xlsx` |
| **+ 新建枚举** | 弹出对话框，输入枚举名和枚举值，GUI 自动写入 `__enums__.xlsx` |
| **+ 新建 Bean** | 弹出对话框，输入 Bean 名和字段，GUI 自动写入 `__beans__.xlsx` |
| **📥 导入文件** | 打开文件选择对话框，将已有 xlsx 注册到 `__tables__.xlsx` |

> **拖入方式**：用户也可将 xlsx 直接拖入 `Datas/` 目录，然后点击「刷新表格列表」(`F5`)，GUI 会提示是否注册到 `__tables__.xlsx`。

### 主内容区

水平分为左右两个面板，中间有可拖动分隔条。

#### 左侧：表格列表面板

```
┌──────────────────────┐
│ 📋 表格列表           │
│──────────────────────│
│ 🔍 [过滤...________] │
│                      │
│ ▾ 全部 (5)           │
│   ├ Item_Config   ✓  │
│   ├ Skill_Config  ✓  │
│   ├ Drop_Table    ✓  │
│   ├ Level_Config  ✓  │
│   └ NPC_Config    ✗  │
└──────────────────────┘
```

**功能**：

- **过滤框**：实时按名称筛选
- **状态图标**：`✓` 绿色（成功）/ `✗` 红色（失败）/ `—` 灰色（未导表）
- **单击**：右侧面板显示该表预览
- **右键菜单**：
  - 单独导出此表
  - 在文件管理器中显示源文件
  - 在系统程序中打开源文件
  - 复制表格名称
  - 从项目中移除（仅移除注册信息，不删除文件）

> 未打开项目时，列表显示提示："请先在「文件」中新建或打开一个项目"。

---

#### 右侧：表格内容面板

```
┌──────────────────────────────────────────────────────────────────────┐
│ 📄 Item_Config                                    [↗ 在编辑器中打开] │
│──────────────────────────────────────────────────────────────────────│
│  ┌────┬──────────┬─────────┬──────────────────────────────────────┐  │
│  │ id │ name     │  value  │ desc                                 │  │
│  ├────┼──────────┼─────────┼──────────────────────────────────────┤  │
│  │  1 │ item_001 │     100 │ 基础道具                             │  │
│  │  2 │ item_002 │     200 │ 进阶道具                             │  │
│  └────┴──────────┴─────────┴──────────────────────────────────────┘  │
│  共 128 行                                                           │
└──────────────────────────────────────────────────────────────────────┘
```

**功能**：

- **表头排序**：点击列标题按该列升/降序排列
- **行数统计**：底部显示总行数
- **虚拟化渲染**：支持大量数据行不卡顿
- **双击列标题**：打开源 Excel 文件，并通过 COM 自动化定位到对应列（Windows + Excel 环境）；其他环境退化为仅打开文件
- **[↗ 在编辑器中打开]**：用系统默认程序打开源文件
- **无选中时**：显示提示"← 请在左侧列表中选择一张表格"
- **读取失败时**：显示错误信息（如"文件不存在"、"格式错误"）

## 三、二级窗口

### 导出配置窗口

通过「操作 > 导出配置...」(`Ctrl+,`) 打开的模态对话框。

```
┌──────────────────── 导出配置 ───────────────────────────────────────┐
│                                                                     │
│  项目: MyGame                                                       │
│                                                                     │
│  ▾ 数据导出 ──────────────────────────────────────────────────     │
│    导出类型:    [bin               ▾]                               │
│    输出路径:    [__________________________________] [📂 浏览]      │
│                                                                     │
│  ▾ 代码导出 ──────────────────────────────────────────────────     │
│    ☑ 生成代码                                                       │
│    代码类型:    [cs-bin            ▾]                               │
│    输出路径:    [__________________________________] [📂 浏览]      │
│    顶层命名空间: [cfg_________________________________]             │
│                                                                     │
│  ▾ 导出目标 ──────────────────────────────────────────────────     │
│    目标:        [all               ▾]                               │
│                  说明: all = 导出 client + server 两端数据          │
│                                                                     │
│                                       [取消]  [保存并关闭]          │
└─────────────────────────────────────────────────────────────────────┘
```

#### 数据导出区

| 字段 | 说明 | 控件类型 |
|------|------|---------|
| 导出类型 | 数据序列化格式，如 `bin`、`json`、`lua` | 下拉选择 |
| 输出路径 | 导出数据文件的目标目录 | 路径输入框 + 浏览按钮 |

#### 代码导出区

| 字段 | 说明 | 控件类型 |
|------|------|---------|
| 生成代码 | 是否同时生成代码文件 | 复选框（不勾选时禁用下方字段） |
| 代码类型 | 如 `cs-bin`、`typescript-json`、`java-bin` | 下拉选择 |
| 输出路径 | 生成代码文件的目标目录 | 路径输入框 + 浏览按钮 |
| 顶层命名空间 | 生成代码的顶层模块名，如 `cfg` | 文本框 |

#### 导出目标区

| 字段 | 说明 | 控件类型 |
|------|------|---------|
| 目标 | 对应 `luban.conf` 中定义的 targets，如 `all`、`client`、`server` | 下拉选择（从 luban.conf 动态读取） |

> 所有 Luban CLI 底层参数（`--conf`、`--target`、`--codeTarget`、`--dataTarget`、`--xargs` 等）均由 GUI 自动组装，**用户不需要了解也不需要填写任何命令行参数**。

### 日志窗口

通过「视图 > 日志窗口」(`Ctrl+L`) 打开的独立浮动窗口。关闭后日志内容保留，重新打开时仍可查看本次会话历史。

```
┌──────────────────── 日志 ──────────────────────────────────── [─][□][×]┐
│                                                         [🗑 清空]      │
│  [INFO]    15:00:01  就绪，等待导表...                                  │
│  [INFO]    15:00:05  启动 Luban 进程...                                 │
│  [OUT]               Processing Excel files...                         │
│  [OUT]               Sheet1: 100 rows                                  │
│  [WARN]              Missing optional field at Sheet2!B3               │
│  [SUCCESS] 15:00:07  导表完成 (耗时 2.34s)                              │
│                                                                        │
│                                             [↓ 跳到最新]               │
└────────────────────────────────────────────────────────────────────────┘
```

#### 日志级别与颜色

| 前缀 | 颜色 | 含义 |
|------|------|------|
| `[INFO]` | 灰色 | GUI 自身产生的流程信息 |
| `[OUT]` | 白色/默认 | Luban stdout 原始输出 |
| `[WARN]` | 黄色 | 警告（Luban stderr 或 GUI 判断） |
| `[ERROR]` | 红色 | 错误信息 |
| `[SUCCESS]` | 绿色 | 导表成功（进程退出码为 0 时追加） |

#### 窗口功能

- **自动弹出**：每次执行"全量导表"或"校验配置"时，若未打开则自动打开
- **自动滚动**：默认跟随最新日志；手动向上滚动时暂停，出现"↓ 跳到最新"按钮
- **清空**：点击 🗑 按钮清空；每次新导表启动前自动清空
- **全选 + 复制**：支持 `Ctrl+A` 全选、`Ctrl+C` 复制
- **虚拟化渲染**：支持 10,000+ 条日志不卡顿

## 五、交互行为规范

### 菜单项状态

| 状态 | 全量导表 | 校验配置 | 取消 |
|------|---------|---------|------|
| 初始（未打开项目） | 置灰 | 置灰 | 置灰 |
| 就绪（项目已打开，配置已填） | 可用 | 可用 | 置灰 |
| 导表进行中 | 置灰 | 置灰 | 可用 |
| 导表完成 | 可用 | 可用 | 置灰 |

### 路径输入框行为

- 支持直接拖拽文件夹到输入框
- 路径不存在时，输入框边框高亮为红色，Tooltip 显示具体错误
- 路径有效时，边框为默认色

### 双击列标题（定位到 Excel）

1. 获取该列对应的字段名和源 Excel 文件路径
2. 调用 `FileOpenService.OpenFileAtCell()`
3. **Windows + Excel**：COM 自动化打开并定位到对应列
4. **其他环境**：退化为用默认程序打开 Excel 文件

### 校验配置反馈

- **通过**：状态区显示绿色 `✓ 配置有效`，3 秒后恢复；日志窗口自动弹出
- **失败**：状态区显示红色 `✗ 配置错误`；日志窗口弹出并追加错误列表：
  ```
  [ERROR] 校验失败，发现以下问题：
  [ERROR]   • 数据输出路径为空
  [ERROR]   • 代码输出路径不存在: D:\NoSuch\Scripts
  ```

### 首次启动引导

无已注册项目时，弹出提示，引导用户新建或打开项目。

## 七、后续版本规划

### v1.1：多导出配置预设

在项目切换栏中增加**导出配置预设**管理，支持同一项目下保存多套导出配置（如"出 client bin"和"出 server json"预设），快速切换。

### v1.2：日志增强

- 日志窗口顶部增加过滤工具栏（按级别筛选、关键字搜索）
- 支持将日志导出为文本文件
- 显示导表历史记录（可点击查看历史日志）

### v2.0：在线字段编辑

在右侧面板中直接编辑表格数据并写回 xlsx，操作样式参考轻量级电子表格（禁止删除 `##` 元数据行）。
