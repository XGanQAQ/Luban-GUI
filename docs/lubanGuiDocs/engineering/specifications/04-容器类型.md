## 一、容器类型概览

容器类型用于表示一个字段包含多个元素。在 `__beans__.xlsx` 的 `*fields` 子表格中，将字段 `type` 列写成如下格式：

| 容器类型 | 类型写法（`type` 列） | 对应 C# 类型 | 约束 |
|----------|-----------------------|--------------|------|
| `array` | `array,<元素类型>` | `T[]` | 元素类型不能为可空 |
| `list` | `list,<元素类型>` | `List<T>` | 元素类型不能为可空 |
| `set` | `set,<元素类型>` | `HashSet<T>` | 元素类型不能为可空；元素值必须唯一 |
| `map` | `map,<键类型>,<值类型>` | `Dictionary<K,V>` | 键只能为基本类型或枚举；键/值均不能为可空；键值必须唯一 |

**常见写法示例：**

```
array,int           // int 数组
list,string         // 字符串列表
set,Color           // Color 枚举集合
map,int,string      // int→string 字典
map,int,item.Item   // int→Item 结构体字典
list,item.Item      // Item 结构体列表
```

> **注意**：容器类型本身**不支持可空**（即不能写 `list,int?`），但容器的元素类型也不允许为可空。

## 二、在数据 xlsx 中填写容器字段

### 2.1 多单元格（合并列）流式格式

将容器字段名的标题单元格**横向合并**，覆盖容器所有元素所需的列数，然后按顺序每个单元格填一个元素。空白单元格会被跳过忽略。

**array/list/set 示例**（`skills` 字段类型 `list,int`，合并 C1:E1）：

```
##var   id      skills（C1:E1 合并）
##type  int     list,int
##      ID      技能列表
(空)    1001    100    200    300
(空)    1002    50     (空)   400    ← 中间空列被跳过，等同 [50, 400]
```

**map 示例**（`rewards` 字段类型 `map,int,int`，合并多列），key-value 交替填写：

```
##var   id      rewards（C1:H1 合并）
##type  int     map,int,int
(空)    1001    1001   10   1002   5    ← key=1001,val=10; key=1002,val=5
```

**元素为结构体 示例**（`items` 字段类型 `list,item.Item`，Item 有 id/count/desc 三个字段）：

```
##var   id      items（合并多列，每个 Item 占 3 列）
##type  int     list,item.Item
(空)    1001    101  10  短剑   201  5  弓箭         ← 两个 Item 元素
```

---

### 2.2 单单元格 sep 分割格式

在 `type` 栏的容器类型上加 `#sep=<分隔符>` 标签，整个容器在**一个单元格**里用分隔符分开每个元素。

**array/list/set 示例**（`sep=,`）：

```
##var   id      skills
##type  int     (list#sep=,),int
(空)    1001    100,200,300
(空)    1002    50,400
```

**map 示例**（通常需要两种分隔符：元素间用一个，key/value 间用另一个）：

```
##var   id      rewards
##type  int     (map#sep=|,),int,string
(空)    1001    1001|金币,1002|经验        ← key=1001,val=金币; key=1002,val=经验
```

> 第二个 sep 字符（`,`）分割 key-value 对，第一个 sep 字符（`|`）分割 key 与 value。  
> `sep` 支持多个字符，每个字符都是独立的分隔符（不是整体字符串作为分隔符）。

**元素为结构体的 sep 示例**（Item 本身需有 `sep=,`，外层用 `;` 分割元素）：

```
##var   id      items
##type  int     (list#sep=;),item.Item     ← Item 定义了 sep=,
(空)    1001    101,10,短剑;201,5,弓箭
```

### 2.3 多行模式（`*fieldName`）

字段名前加 `*`，则容器每个元素单独占一条数据行。适合元素较多或元素本身字段较多的情况。  
**新记录的起始判断**：某个 `non_empty` 字段（如 `id`）出现新值时，开始新的父记录。

**array/list 多行示例**（`*skills` 字段）：

```
##var   id      *skills
##type  int     list,int
(空)    1001    100
(空)    (空)    200
(空)    (空)    300        ← id 列为空，继续追加到 id=1001 的 skills
(空)    1002    50
(空)    (空)    400
```

**map 多行示例**（key/value 在同一行里横向排列）：

```
##var   id      *rewards（合并两列）
##type  int     map,int,int
(空)    1001    1001   10
(空)    (空)    1002   5
(空)    1002    2001   100
```

**list of bean 多行 + 列限定示例**（`*items`，元素类型 `list,item.Item`，同时用多级标题头限定子字段）：

```
行1 ##var   id      *items（合并多列）
行2 ##var   (空)    item_id   count   desc
##type      int     list,item.Item
(空)        1001    101       10      短剑
(空)        (空)    201       5       弓箭
(空)        1002    301       1       魔杖
```

---

### 2.4 列限定格式（多级标题头）

在第一行 `##var` 写容器字段名（合并列），在下一行 `##var` 写容器**元素**的子字段名，从而为每个子字段分配固定列。  
适合元素为 bean 且字段较多的场景，可以避免流式填写错列。**列限定下空单元格可取默认值**。

```
行1 ##var   id      items（C1:E1 合并）
行2 ##var   (空)    item_id   count   desc
##type      int     list,item.Item
(空)        1001    101       10      短剑
(空)        1001    201       5       弓箭      ← 注意：多行模式须配合 *items
```

> 纯列限定（不加 `*`）下，容器字段只能在同一行内流式填写所有元素；  
> 列限定通常配合多行模式（`*items`）使用，让每个元素独占一行且子字段有明确列。

**map 列限定（`$key` 子列）示例**：

```
行1 ##var   id      *rewards（合并多列）
行2 ##var   (空)    $key    value
##type      int     map,int,int
(空)        1001    1001    10
(空)        (空)    1002    5
```

## 三、各填法对比

| 填法 | 适用场景 | 支持默认值 | 备注 |
|------|----------|------------|------|
| 多单元格合并（流式） | 元素较少，字段简单 | ❌ | 空格被跳过，不支持默认值 |
| 单单元格 `#sep=` | 元素少、紧凑 | ❌ | 需在 `type` 栏写容器分隔符（如 `(array#sep=;),int`） |
| 多行模式 `*field` | 元素较多 | ❌（流式子读取）/ ✅（列限定子读取） | 字段名加 `*` 前缀 |
| 多级标题头（列限定） | 元素为 bean 且字段多 | ✅ | 需新增 `##var` 子行 |

## 四、注意事项

1. **容器类型不可为可空**：`list,int?`、`array,string?` 均不合法。  
2. **map 的 key 类型限制**：只能是基本类型（`int`、`string` 等）或枚举类型，不能是 bean。  
3. **set 的元素唯一性**：Luban 会在加载时校验 set 的元素是否重复，重复会报错。  
4. **流式格式下空单元格被跳过**：使用多单元格流式或 sep 分割时，**不能依赖空单元格表示默认值**，所有元素必须显式填写。  
5. **多行模式的记录边界**：使用 `*field` 时，新记录以主键字段（non_empty 约束的字段，通常是 `id`）出现新值为标志。确保 id 列在新记录首行填写，其余行留空。
