## 一、枚举（enum）

### 1.1 在 `__enums__.xlsx` 中定义枚举

每一条枚举记录占多个数据行（多行模式），顶层字段只在第一行填写，每个枚举项单独占一行。

#### 顶层字段（B~G 列）

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `full_name` | 枚举全名，含模块前缀（如无模块直接写类型名） | `item.Color`、`AccessFlag` |
| `flags` | 是否为标志位枚举。`true` = 可多个值按位组合；`false` = 普通枚举 | `false` |
| `unique` | 枚举项的整数值是否不允许重复。建议填 `true` | `true` |
| `group` | 导出分组，空=全部分组，否则填 `c`/`s`/`e` 等 | 空 |
| `comment` | 注释 | `颜色枚举` |
| `tags` | 自定义标签（键值对格式，一般留空） | 空 |

#### `*items` 子字段（H~L 列，H1:L1 合并单元格）

每个枚举项独占一行数据行，`full_name` 等顶层字段只在第一行填写，后续行留空。

| 子字段名 | 说明 | 示例 |
|----------|------|------|
| `name` | 枚举项名（英文标识符）| `RED`、`READ` |
| `alias` | 别名（中文或简写，策划可用别名填数据） | `红`、`读` |
| `value` | 枚举整数值；留空则自动从上一项递增（首项默认 0）；也可填 `0x1`（十六进制）或 `A\|B`（位组合） | `1`、`0x01` |
| `comment` | 注释 | `红色` |
| `tags` | 标签（一般留空） | 空 |

#### 示例 1：普通枚举 `Color`

```
full_name   flags   unique   group   comment   tags   [*items]
                                                       name     alias   value   comment   tags
item.Color  false   true     (空)    颜色枚举   (空)   RED      红      1       红色      (空)
(空)        (空)    (空)     (空)    (空)       (空)   GREEN    绿      2       绿色      (空)
(空)        (空)    (空)     (空)    (空)       (空)   BLUE     蓝      3       蓝色      (空)
```

> **提示**：`value` 留空时自动从 0 开始递增，所以如果枚举项值正好是 0, 1, 2... 可省略不填。

#### 示例 2：标志位枚举 `AccessFlag`（`flags=true`）

```
full_name    flags   unique   group   comment     tags   [*items]
                                                          name      alias   value   comment   tags
AccessFlag   true    true     (空)    访问权限    (空)   READ      读      0x01    读权限    (空)
(空)         (空)    (空)     (空)    (空)        (空)   WRITE     写      0x02    写权限    (空)
(空)         (空)    (空)     (空)    (空)        (空)   EXECUTE   执行    0x04    执行权限  (空)
```

### 1.2 在数据表中填写枚举字段

在数据 xlsx 的 `##type` 行，字段类型直接填枚举 `full_name`（可省略模块前缀，Luban 会自动查找）。

#### 普通枚举的填法

以下三种写法等价，在单元格里任填其一：

| 写法 | 含义 |
|------|------|
| `RED` | 枚举项名 |
| `红` | 枚举项别名（alias） |
| `1` | 枚举项对应整数值 |

> 如果枚举有值为 0 的项，单元格**留空**自动取该项；否则留空会报错。

#### 标志位枚举的填法

使用 `|` 组合多个枚举项（也可使用别名或值）：

| 写法 | 含义 |
|------|------|
| `READ` | 单个标志位 |
| `READ\|WRITE` | 按位或组合 |
| `读\|写` | 用别名组合 |
| `0x01\|0x02` | 用值组合 |

> 如果在 `__enums__.xlsx` 中为该枚举配置了 `sep`（如 `sep=,`），则改用 `,` 作为分割符：`READ,WRITE`。

## 二、结构体（bean）

### 2.1 在 `__beans__.xlsx` 中定义结构体

每一条 bean 记录占多个数据行（多行模式），顶层字段只在第一行填写，每个成员字段（field）单独占一行。

#### 顶层字段（B~I 列）

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `full_name` | Bean 全名，含模块前缀 | `item.Item`、`MyVec3` |
| `parent` | 父类全名（多态继承时填写，无继承留空） | `Shape`、空 |
| `valueType` | 是否为值类型（生成 struct 而非 class），填 `true`/`false`/留空 | 空 |
| `sep` | 紧凑模式下的字段分割符；在一个单元格内填写整个结构时使用 | `,`、`;` |
| `alias` | 别名，策划可用中文名指定多态类型 | `圆`、空 |
| `comment` | 注释 | `道具结构体` |
| `group` | 导出分组，空=全部分组 | 空 |
| `tags` | 自定义标签（一般留空） | 空 |

#### `*fields` 子字段（J~P 列，J1:P1 合并单元格）

每个成员字段独占一行数据行，`full_name` 等顶层字段只在第一行填写，后续行留空。

| 子字段名 | 说明 | 示例 |
|----------|------|------|
| `name` | 字段名（英文） | `id`、`count` |
| `alias` | 字段别名 | `编号`、空 |
| `type` | 字段类型（见下方类型说明） | `int`、`string`、`item.Color`、`list,int` |
| `group` | 分组，空=全部分组 | 空 |
| `comment` | 注释 | `道具ID` |
| `tags` | 标签（如校验器 `ref=item.TbItem`） | 空 |
| `variants` | 字段变体（高级特性，一般留空） | 空 |

**type 字段的写法：**

| 类型 | 写法示例 |
|------|---------|
| 基本类型 | `int`、`string`、`bool`、`float`、`long` |
| 可空基本类型 | `int?`、`string?` |
| 枚举类型 | `item.Color`（完整名）或 `Color`（同模块下可省略前缀） |
| 嵌套 bean | `item.Item`、`MyVec3` |
| 容器类型 | `list,int`、`array,string`、`set,Color`、`map,int,string` |
| 紧凑嵌套 | `item.Item#sep=,`（临时覆盖 sep，仅此字段生效） |

---

```
item.Item   (空)    (空)       (空)  (空)   道具结构  (空)   (空)  id        (空)   int     (空)   道具ID    (空)  (空)
(空)        (空)    (空)       (空)  (空)   (空)      (空)   (空)  count     (空)   int     (空)   数量      (空)  (空)
(空)        (空)    (空)       (空)  (空)   (空)      (空)   (空)  desc      (空)   string  (空)   描述      (空)  (空)
```

#### 示例 2：紧凑结构体 `MyVec3`（`sep=,`）

```
full_name   parent  valueType  sep   alias  comment   group  tags  [*fields]
                                                                     name  alias  type   group  comment  tags  variants
MyVec3      (空)    (空)       ,     (空)   三维坐标  (空)   (空)  x     (空)   float  (空)   X坐标    (空)  (空)

```

```
Shape      (空)    (空)  (空)   抽象形状父类  （无字段，或定义公共字段）
```

### 2.2 在数据表中填写结构体字段

#### 方式一：多单元格（合并列）流式格式（推荐）

将字段名单元格横向合并覆盖结构所需的列数，子字段按顺序横向排列。

```
##var   id    item（C1:E1 合并）
##type  int   Item
##      ID    道具
(空)    1001  1            10     描述1
(空)    1002  2            5      描述2
              ↑item_id  ↑count  ↑desc
```

> **注意（流式格式）**：空白单元格会被跳过，**不能用空单元格表示默认值**，该填的字段必须逐一填写。

#### 方式二：紧凑单元格（sep 分割）

在 Bean 定义里设置了 `sep`，或者在字段名标签中指定 `#sep=,`，则一个单元格内用分割符隔开所有字段值。

```
##var   id    pos#sep=,
##type  int   MyVec3
(空)    1001  1.0,2.0,3.0
(空)    1002  4.0,5.0,6.0
```

> **注意（sep/紧凑格式）**：sep 分割出的空字符串视为空值，**不支持默认值**，必须显式填写每个字段。

#### 方式三：多级标题头（列限定格式，推荐用于字段多的结构）

在第一行 `##var` 写父字段名（合并对应列），在下一行 `##var` 写子字段名。这样每个子字段有明确的列，不容易出错，且**支持空单元格取默认值**。

```
行1 ##var   id      item（C1:E1 合并）
行2 ##var   (空)    item_id   count   desc
##type      int     int       int     string
(空)        1001    101       10      (留空→默认"")
```

#### 方式四：多态 bean 的填写

多态字段在数据中须先填写**子类名**（或子类别名），再按顺序填写子类字段。

- **流式格式**（无 `$type` 列）：

```
##var    id    shape（合并多列）
##type   int   Shape
(空)     1     Circle   5.0         ← 先写子类名 Circle，再写 radius=5.0
(空)     2     Rect     8.0   4.0   ← 子类名 Rect，再写 width=8.0, height=4.0
(空)     3     圆       3.0         ← 也可用别名
```

- **列限定格式**（使用 `$type` 列）：

```
行1 ##var   id      shape（合并）
行2 ##var   (空)    $type    radius   width   height
(空)        1       Circle   5.0      (空)     (空)
(空)        2       Rect     (空)     8.0      4.0
```

> 可空多态 bean（`Shape?`）中，`$type` 填 `null` 表示空值。

## 三、注意事项

1. **`full_name` 的模块前缀**：在 `__beans__.xlsx` / `__enums__.xlsx` 中，`full_name` 必须与 `luban.conf` 中 `schemaFiles` 配置的 `type` 对应。不含模块时直接写类型名；含模块时写 `模块名.类型名`，如 `item.Color`。

2. **`__beans__.xlsx` 内置类型**：`Defines/builtin.xml` 中已预定义 `vector2`、`vector3`、`vector4` 等常用类型，无需重复添加。

3. **流式格式不支持默认值**：流式格式（多单元格或 sep 分割）下，空单元格会被忽略跳过，必须显式填写所有字段。列限定（多级标题头）格式下才支持空单元格取默认值。

4. **多行模式中的顶层字段**：`__beans__` / `__enums__` 使用多行模式（`*fields`/`*items`），同一个 bean/enum 记录的第二行起，顶层字段（`full_name` 等）必须**留空**，否则会被视为新记录。
